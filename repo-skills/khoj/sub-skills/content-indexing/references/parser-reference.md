# Parser Reference

Khoj parser classes live under `khoj.processor.content`. Use static extraction methods for parser-only debugging. Avoid instance `process()` methods unless intentionally writing to a configured Khoj database, because `process()` calls `update_embeddings()` and creates/deletes entries and file objects.

## Entry Shape

Parsers return `khoj.utils.rawconfig.Entry` objects with these useful fields:

- `raw`: source text for the parsed chunk.
- `compiled`: text sent to embedding/search. Parsers usually prepend filename and heading context.
- `heading`: heading or filename context.
- `file`: source filename, path, or URL.
- `uri`: explicit URI when available; otherwise `Entry` derives `file://{file}` for local files or keeps HTTP URLs.
- `corpus_id`: a string UUID for chunks split from the same original entry; may be `None` before token splitting.

`Entry.to_json()` serializes the object dictionary. `Entry.from_dict()` rebuilds an `Entry` from fields used in JSON-style fixtures.

## Parser Classes and Safe Static APIs

### Markdown

Class: `MarkdownToEntries`

Safe parser methods:

- `extract_markdown_entries(markdown_files: dict[str, str], max_tokens=256) -> (file_to_text_map, entries)`
- `process_single_markdown_file(markdown_content, markdown_file, entries, entry_to_file_map, max_tokens=256, ancestry={}, start_line=1)`
- `convert_markdown_entries_to_maps(parsed_entries, entry_to_file_map) -> list[Entry]`

Behavior:

- Splits by Markdown heading hierarchy when a section exceeds `max_tokens`.
- Prepends heading ancestry to child chunks.
- Preserves intro text before headings as its own entry.
- `compiled` is prefixed with the filename as top-level context.
- Local-file URIs include `file://{filename}#line={line}`. HTTP filenames remain URL-like URIs.

### Org

Class: `OrgToEntries`

Safe parser methods:

- `extract_org_entries(org_files: dict[str, str], index_heading_entries=False, max_tokens=256) -> (file_to_text_map, entries)`
- `extract_org_nodes(org_files, max_tokens) -> (file_to_text_map, node_groups, entry_to_file_map)`
- `process_single_org_file(org_content, org_file, entries, entry_to_file_map, max_tokens=256, ancestry={}, start_line=1)`
- `convert_org_nodes_to_entries(parsed_entries, entry_to_file_map, index_heading_entries=False) -> list[Entry]`

Behavior:

- Splits recursively by Org heading levels when a tree exceeds `max_tokens`.
- Ignores heading-only entries by default unless `index_heading_entries=True`.
- Adds filename and ancestor trail to compiled headings.
- Parses tags, TODO state, closed date, scheduled date, body text, and node properties.
- URI can come from the Org node `LINE` property generated by the parser.

### Plaintext, HTML, XML

Class: `PlaintextToEntries`

Safe parser methods:

- `extract_plaintext_entries(text_files: dict[str, str]) -> (file_to_text_map, entries)`
- `process_single_plaintext_file(text_content, text_file, entries, entry_to_file_map)`
- `convert_text_files_to_entries(parsed_entries, entry_to_file_map) -> list[Entry]`
- `extract_html_content(markup_content, markup_type)`

Behavior:

- Treats each plaintext file as one entry before token splitting.
- For filenames ending in `html`, `htm`, or `xml`, strips markup with BeautifulSoup and indexes visible text.
- `compiled` is `{filename}\n{raw_text}` and `heading` is the filename.

### PDF

Class: `PdfToEntries`

Safe parser methods:

- `extract_pdf_entries(pdf_files: dict[str, bytes]) -> (file_to_text_map, entries)`
- `extract_text(pdf_file: bytes) -> list[str]`
- `convert_pdf_entries_to_maps(parsed_entries, entry_to_file_map) -> list[Entry]`
- `clean_text(text)` removes null bytes.

Behavior:

- Uses `PyMuPDFLoader` through a temporary `.pdf` file.
- Produces page-level text entries before token splitting.
- `compiled` and `heading` include the source filename.
- OCR behavior is not guaranteed by the basic PDF parser; scanned PDFs may need optional OCR paths or may extract little text.

### DOCX

Class: `DocxToEntries`

Safe parser methods:

- `extract_docx_entries(docx_files: dict[str, bytes]) -> (file_to_text_map, entries)`
- `extract_text(docx_file: bytes) -> list[str]`
- `convert_docx_entries_to_maps(parsed_entries, entry_to_file_map) -> list[Entry]`

Behavior:

- Uses `Docx2txtLoader` through a temporary `.docx` file.
- Usually emits one or more text blobs from the document loader.
- `compiled` and `heading` include the source filename.

### Images

Class: `ImageToEntries`

Safe parser methods:

- `extract_image_entries(image_files: dict[str, bytes]) -> (file_to_text_map, entries)`
- `convert_image_entries_to_maps(parsed_entries, entry_to_file_map) -> list[Entry]`

Behavior:

- Accepts `.png`, `.jpg`, `.jpeg`, and `.webp` filenames.
- Uses `rapidocr_onnxruntime.RapidOCR` when installed.
- Missing OCR dependency logs a warning and skips the image.
- Writes a temporary image file in the current working directory and removes it afterward.

### GitHub

Class: `GithubToEntries`

Remote-source behavior:

- Constructor expects a stored `GithubConfig` model with `pat_token` and related repository rows.
- `process()` warns if the PAT is missing, downloads each repository, and updates database entries.
- `get_files()` reads the recursive Git tree for the configured branch, collects Markdown, Org, and Magika-detected text/code blobs.
- Static helpers reuse the local Markdown/Org/plaintext parser conversion paths for downloaded file contents and GitHub blob URLs.
- Rate-limit exhaustion raises `ConnectionAbortedError`.

Use `/api/content/github` to store the config. Do not pass GitHub repo data to local multipart upload unless intentionally indexing exported files as local content.

### Notion

Class: `NotionToEntries`

Remote-source behavior:

- Constructor expects a stored `NotionConfig` model with `token`.
- `process()` calls Notion `/v1/search`, paginates with `start_cursor`, then processes pages.
- Page title fields are selected from `title`, `Title`, `Name`, `Page`, or `Event`.
- Display block text includes paragraphs, headings, list items, to-dos, toggles, and child pages.
- Several unsupported/non-text block types are skipped.
- Links are converted to `<a href='...'>plain_text</a>` strings.

Use `/api/content/notion` to store the token. The Notion integration must be shared with pages/workspaces before pages appear in search results.

## Chunking and Line Numbers

`TextToEntries.split_entries_by_max_tokens(entries, max_tokens=256, max_word_length=500, raw_is_compiled=False)` performs common chunking:

- Drops entries whose `compiled` text is empty.
- Splits with preference order: paragraphs, lines, `!`, `?`, `.`, spaces, tabs, then characters.
- Uses no chunk overlap.
- Gives chunks from the same source entry a shared `corpus_id`.
- For chunks after the first, prepends the last 100 characters of `heading` when available.
- Removes words longer than `max_word_length` from compiled text.
- Removes null characters from `compiled`, `raw`, `heading`, `file`, and `uri` fields.
- If `entry.uri` starts with `file://` and contains `#line=`, it adjusts the line number per chunk by finding the chunk text in `entry.raw`.
- If `raw_is_compiled=True`, chunk `raw` is set to the compiled chunk text. Plaintext processing uses this mode after extraction.

## Deletion Markers and Process Methods

Each local `process(files, user, regenerate=False)` method computes deletion candidates before parsing:

- Markdown, Org, and plaintext treat `files[name] == ""` as a delete marker.
- PDF, DOCX, and image treat `files[name] == b""` as a delete marker.
- Non-empty files are parsed and embedded.
- Deletion filenames are passed to `update_embeddings()` to delete matching entries and file objects.
- `regenerate=True` deletes all entries of the selected file type before adding current entries.

Do not emulate deletion by calling parser static methods; static methods only parse input and return entries.

## Parser-Only Helper

Use `scripts/parse_content_fixture.py` for tiny parser checks. It supports Markdown, Org, and plaintext strings or files and prints JSON without database writes.

Examples:

```bash
python skills/khoj/sub-skills/content-indexing/scripts/parse_content_fixture.py --type markdown --text '# Project\nDetails' --name notes.md
python skills/khoj/sub-skills/content-indexing/scripts/parse_content_fixture.py --type org --text '* Task\nBody' --max-tokens 8
python skills/khoj/sub-skills/content-indexing/scripts/parse_content_fixture.py --type plaintext --file sample.txt
```

The helper intentionally does not support PDF, DOCX, image, GitHub, or Notion parsing because those paths may require optional binary/OCR/network dependencies or stored server config.
