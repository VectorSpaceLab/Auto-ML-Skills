# pymatgen Configuration Reference

Pymatgen reads settings from environment variables and a YAML configuration file commonly described as `.pmgrc.yaml`. Public docs recommend persistent settings with `pmg config --add VARIABLE_NAME VALUE`, but that command rewrites the config file, so treat it as mutating.

## Safe Configuration Policy

Before changing configuration, confirm:

1. Exact variable names and values.
2. Whether credentials should be stored persistently or used only in the current session.
3. Backup suffix for `pmg config --add`; default `.bak` is safest.
4. Whether path values exist and are intended for pymatgen use.
5. For POTCAR setup, that the user has licensed VASP pseudopotentials and approves copying/reformatting them.
6. For CP2K data setup, that the user approves generating many YAML resource files from CP2K basis/potential files.

Prefer help and redacted probes first:

```bash
pmg config --help
python scripts/pymatgen_environment_probe.py
```

## Config File Behavior

`pmg config --add` chooses the current pymatgen settings file if it exists, otherwise an older settings file if present, otherwise it creates the current settings file. Existing config files are backed up before rewrite when `--backup` is non-empty.

- Default backup: `pmg config --backup .bak --add KEY VALUE`.
- Custom backup: `pmg config --backup .before-pmg-change --add KEY VALUE`.
- Disable backup: `pmg config --backup '' --add KEY VALUE`; avoid unless explicitly requested.
- Type conversion: values `true`, `false`, `none`, and `null` are converted to booleans/null by the CLI handler.
- Config override: `PMG_CONFIG_FILE` can point pymatgen at a custom config-file location for an isolated session. Do not publish local config paths in reusable skill content.

Never print secret values back to the user. Redact API keys in summaries and command transcripts.

## Public Variables

| Variable | Purpose | Safe Guidance |
| --- | --- | --- |
| `PMG_CONFIG_FILE` | Environment variable selecting a custom config file. | Useful for isolated tests or temporary sessions. Avoid writing machine-specific paths into shared instructions. |
| `PMG_MAPI_KEY` | Materials Project API key used by `MPRester` and `pmg query`. | Prefer session-scoped environment variables for one-off work; persist only with approval. Route live query details to `external-data-access`. |
| `PMG_VASP_PSP_DIR` | Directory where pymatgen looks for VASP pseudopotential resources. | Set only after user-approved POTCAR setup or when the user provides an existing pymatgen-compatible resource directory. |
| `PMG_POTCAR_CHECKS` | If false, disables all POTCAR checks, including compatibility and existence checks in VASP I/O. | Do not recommend disabling globally unless the user understands validation and comparability risks. |
| `PMG_DEFAULT_FUNCTIONAL` | Default VASP POTCAR functional; `pmg potcar` uses this, then falls back to `PBE`. | Set only when the user's POTCAR resource directory supports that functional. |
| `PMG_CP2K_DATA_DIR` | Directory for CP2K resource data generated by `pmg config --cp2k`. | Requires user-approved CP2K data conversion. |
| `PMG_DEFAULT_CP2K_FUNCTIONAL` | Default CP2K functional. | Use only after confirming CP2K workflow expectations. |
| `PMG_DEFAULT_CP2K_BASIS_TYPE` | Default CP2K basis type. | Use only after CP2K data setup and basis availability. |
| `PMG_DEFAULT_CP2K_AUX_BASIS_TYPE` | Default CP2K auxiliary basis type. | Use only after CP2K data setup and auxiliary basis availability. |

## Materials Project Keys

`MPRester` and `pmg query` can use `PMG_MAPI_KEY` when no explicit API key argument is passed.

Safe handling:

- Do not ask the user to paste the key into public chat logs.
- For one-off commands, have the user set `PMG_MAPI_KEY` in their shell/session.
- For persistent setup, ask for approval before running `pmg config --add PMG_MAPI_KEY <redacted-api-key>`.
- If authentication, field, rate-limit, or endpoint behavior is the main issue, use [../../external-data-access/SKILL.md](../../external-data-access/SKILL.md).

## POTCAR Setup Boundaries

Pymatgen cannot distribute VASP pseudopotentials. The user must supply licensed POTCAR files.

`pmg config -p INPUT_PSP_DIR OUTPUT_PSP_DIR`:

- Expects an extracted VASP pseudopotential tree such as `potpaw_PBE`, `potpaw_PBE.54`, `potpaw_LDA`, or related functional directories.
- Creates a pymatgen-compatible resource directory.
- Copies, decompresses, renames, and gzips POTCAR files under the target directory.
- Prompts if the destination directory already exists.
- Recommends setting `PMG_VASP_PSP_DIR` afterward.

Safe sequence:

1. Confirm licensing and user ownership of the POTCAR data.
2. Confirm source and destination directories.
3. Ask whether existing destination contents may be reused/overwritten if prompted.
4. Run `pmg config -p ...` only after approval.
5. Ask before persisting `PMG_VASP_PSP_DIR` with `pmg config --add`.
6. Set `PMG_DEFAULT_FUNCTIONAL` only if the functional is available and intended.

Never bundle, display, synthesize, or scramble real POTCAR contents. The maintainer-only POTCAR scrambler is excluded from runtime execution because POTCAR data is licensed and path-sensitive.

## CP2K Data Setup

`pmg config --cp2k INPUT_DATA_DIR OUTPUT_DATA_DIR`:

- Reads CP2K basis and potential files.
- Generates per-element YAML settings files in the output directory.
- Prompts if the destination directory exists.
- Recommends setting `PMG_CP2K_DATA_DIR`, `PMG_DEFAULT_CP2K_FUNCTIONAL`, `PMG_DEFAULT_CP2K_BASIS_TYPE`, and `PMG_DEFAULT_CP2K_AUX_BASIS_TYPE` afterward.

Ask before running because it writes many files and depends on user-provided CP2K data resources.

## Approved Command Templates

Use placeholders in reusable guidance and replace them only after user approval:

```bash
pmg config --backup .bak --add PMG_MAPI_KEY <redacted-api-key>
pmg config --backup .bak --add PMG_VASP_PSP_DIR <pymatgen-potcar-resource-dir>
pmg config --backup .bak --add PMG_DEFAULT_FUNCTIONAL PBE_54
pmg config --backup .bak --add PMG_CP2K_DATA_DIR <pymatgen-cp2k-data-dir>
```

Avoid this unless the user explicitly accepts the risk:

```bash
pmg config --backup .bak --add PMG_POTCAR_CHECKS false
```

Disabling `PMG_POTCAR_CHECKS` globally can mask missing or incompatible POTCAR metadata. Prefer fixing paths, functional names, or POTCAR metadata first.

## Environment Variables vs Persistent Config

Use environment variables for temporary, CI, or isolated sessions. Use `pmg config --add` for persistent user-level defaults. In both cases, keep local paths and secrets out of public skill content, logs, and generated documentation.
