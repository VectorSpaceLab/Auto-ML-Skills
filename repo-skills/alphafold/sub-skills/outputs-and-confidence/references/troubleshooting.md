# Outputs and Confidence Troubleshooting

## `confidence_json` Raises a Rank Error

Symptom:

```text
The plddt array must be rank 1
```

Cause: `confidence_json` expects a one-dimensional pLDDT array shaped `[num_res]`. Passing batched arrays, per-atom B-factor arrays, or logits directly will fail.

Fix:

- If starting from logits, call `compute_plddt(logits)` first.
- If starting from prediction output, use `prediction_result['plddt']` directly.
- If starting from a structure file, remember that B-factors may be per atom; reduce or map them back to residue-level values only when that is scientifically appropriate.

## pLDDT Category or Range Looks Wrong

Cause: categories are defined only for pLDDT values from `0` to `100`.

Fix:

- Confirm the values are pLDDT, not logits or probabilities.
- Confirm any custom parser preserved numeric values rather than strings with percent signs.
- Use `scripts/inspect_confidence_json.py` to summarize category counts from an existing JSON file.

## `pae_json` Raises a Square-Matrix Error

Symptom:

```text
PAE must be a square matrix
```

Cause: PAE must be rank 2 and shaped `N x N`.

Fix:

- Confirm the input is `predicted_aligned_error`, not aligned-confidence logits shaped `[N, N, bins]`.
- If starting from logits and breaks, call `compute_predicted_aligned_error(logits, breaks)` first and use the returned `predicted_aligned_error` matrix.
- Check for truncated JSON or accidental row filtering.

## No `pae_<model>.json` File Exists

Likely causes:

- The selected model preset did not produce predicted aligned error outputs.
- The run failed before the prediction result included PAE fields.
- The task is inspecting an older or custom output directory.

Fix:

- Check `result_<model>.pkl` only if it is safe and appropriate for the task.
- Check `ranking_debug.json`: pTM/multimer-style outputs are more likely to include PAE than plain monomer outputs.
- Route model preset questions to `../model-config-and-api/` and command construction questions to `../prediction-cli/`.

## Ranked Files Are Present but Confidence Files Are Missing

Cause: the directory may be incomplete, manually copied, or generated by a different pipeline.

Fix:

- Inspect `ranking_debug.json` and `timings.json` to confirm this is a normal local AlphaFold output folder.
- Look for per-model `result_<model>.pkl`, `confidence_<model>.json`, and `pae_<model>.json` files.
- Treat the ranked PDB/mmCIF as structure-only evidence if confidence files are unavailable.

## Output Directory Missing Artifacts After a Failed Run

Interpret missing artifacts by stage:

- `msas/` without `features.pkl`: data pipeline or input/search failure.
- `features.pkl` without model results: model config, parameter, JAX, or accelerator failure.
- Model results without ranked files: structure assembly, confidence writing, mmCIF/PDB conversion, or relaxation/finalization failure.
- Relaxed outputs missing while unrelaxed outputs exist: relaxation was disabled, not selected, or failed.

Do not rerun full inference as a validation step. Route prediction execution and flag repair to `../prediction-cli/`.

## PDB Chain Limit Failure

Symptoms mention the PDB format supporting at most 62 chains or inability to build/write more than 62 chains.

Cause: PDB chain IDs are single characters in AlphaFold's PDB writer: uppercase letters, lowercase letters, and digits.

Fix:

- Use mmCIF for many-chain structures when possible.
- Reduce chain count or split the task when PDB output is mandatory.
- Route input chain-count validation to `../input-data-and-formats/`.

## mmCIF Conversion Changes Residues or Chains

Expected behavior from AlphaFold's `Protein` conversion:

- Non-standard residues become `UNK`.
- Non-standard atoms are ignored.
- Missing residue indices are filled as `UNK` in `_entity_poly_seq` when writing mmCIF.
- Original chain IDs may not be preserved after parse-and-write.
- Single-model PDB/mmCIF inputs are supported; multi-model structures and insertion codes can fail.

Fix:

- Preserve the original structure file when exact metadata, non-standard chemistry, or chain IDs matter.
- Use AlphaFold conversion APIs for AlphaFold-style standard protein outputs, not as a lossless structural file editor.

## AFDB Download or Query Cost Risk

Risk signals:

- Full database download requests.
- GCS recursive copies of proteome shards.
- BigQuery metadata queries across broad tables.
- Storage Transfer Service setup.

Fix:

- Prefer website lookup, premade subsets, manifests, or targeted accessions.
- Warn that full dataset handling requires large storage/filesystem capacity.
- Warn that BigQuery and transfer workflows can incur charges outside free-tier limits.
- Do not run cloud commands unless explicitly requested.

## AlphaFold Server JSON Rejected

Common causes:

- Root value is an object instead of a list.
- JSON contains comments or trailing commas.
- `dialect` is missing or not `alphafoldserver`.
- `version` is not compatible with the fields used.
- Entity dictionaries have multiple entity keys or unsupported entity keys.
- Protein/DNA/RNA sequences use unsupported alphabets.
- Modification, ligand, or ion codes are outside the documented allowlists.
- DNA/RNA modifications use the wrong position key.
- Template indices are 1-based instead of zero-based.

Fix:

- Validate with a strict JSON parser.
- Compare against `references/server-example.json`.
- Use `references/server-json.md` for entity-specific required fields and allowlists.
