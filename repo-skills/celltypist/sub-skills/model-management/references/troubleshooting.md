# Model Management Troubleshooting

Use this when CellTypist model discovery, loading, conversion, subsetting, marker extraction, or offline smoke tests fail.

## Model listing unexpectedly downloads files

Symptoms:

- `models.get_all_models()` starts downloading models.
- `models.models_description(on_the_fly=True)` downloads all models in an empty cache.
- Offline runs hang or fail while trying to reach the CellTypist model server.

Cause:

- `get_all_models()` calls `download_if_required()`, which downloads all models when the active cache has no `.pkl` files.
- `models_description(on_the_fly=True)` calls `get_all_models()` before reading descriptions from cached model pickle files.

Recovery:

- Do not use `get_all_models()` for offline inventory when the cache may be empty.
- Run `python scripts/model_cache_check.py` to list local `.pkl` files and `models.json` without network access.
- Pass explicit local model paths containing `/` to `Model.load()` or `celltypist.annotate()`.

## Default model or description fetches remote index

Symptoms:

- `Model.load()` with no argument fails in an offline environment.
- `models.get_default_model()` or `models.models_description(on_the_fly=False)` tries to retrieve model metadata.
- A fresh cache contains no `models.json`.

Cause:

- Default-model resolution uses `get_models_index()`.
- `models_description(on_the_fly=False)` reads `models.json` and downloads it when absent.

Recovery:

- For offline work, avoid `Model.load()` with no argument.
- Avoid `models_description(on_the_fly=False)` unless `models.json` is already cached or network access is intended.
- If online cache population is allowed, run `models.download_models(model="Name.pkl")` for only the needed built-in model, or `models.download_models()` for all models.
- If offline cache population is required, copy known-good `.pkl` files and optional `models.json` into the active `models.models_path` before running CellTypist.

## `CELLTYPIST_FOLDER` did not change the cache

Symptoms:

- `models.models_path` still points to the old cache.
- Files were written under the default home cache instead of the intended folder.

Cause:

- CellTypist computes `models_path` when `celltypist.models` is imported. Changing `CELLTYPIST_FOLDER` afterward does not recompute module-level paths.

Recovery:

- Set `CELLTYPIST_FOLDER` before importing CellTypist.
- In scripts, set the environment variable before the first `import celltypist` or run a new Python process.
- Use `model_cache_check.py --celltypist-folder PATH` to verify the intended cache location in a fresh process.

## `Model.load` says no such file

Symptoms:

- `FileNotFoundError: No such file: ...`.
- A local model named `model.pkl` exists, but loading by bare filename still fails or triggers inventory work first.

Cause:

- Built-in model-name resolution is different from direct file-path loading.
- A bare filename with no `/` may pass through `get_all_models()` before filesystem lookup.

Recovery:

- Use `Path("model.pkl").resolve().as_posix()` or `"./model.pkl"` for strict local loading.
- Confirm the file exists with `python scripts/model_cache_check.py --verify-model ./model.pkl`.
- If the model should be found by name, copy it into the active `models.models_path` and use the exact filename.

## `Model.load` says invalid model

Symptoms:

- `Invalid model: ...` with a pickle or key error.
- Loading succeeds with `pickle.load()` but not `models.Model.load()`.

Cause:

- CellTypist expects a pickle dictionary with keys `Model`, `Scaler_`, and `description`.
- The classifier and scaler must carry sklearn-compatible fitted attributes.
- The file may be corrupted, compressed, generated by incompatible custom code, or not a CellTypist model pickle.

Recovery:

- Verify the structure with `python scripts/model_cache_check.py --verify-model ./model.pkl`.
- Regenerate a smoke-test fixture with `python scripts/tiny_model_factory.py --output tiny_model.pkl`.
- For real custom models, write via `model.write("path.pkl")` after `celltypist.train()` instead of hand-pickling arbitrary objects.

## Marker extraction cannot find a cell type

Symptom:

- `'<cell type>' is not found. Please provide a valid cell type name`.

Cause:

- `extract_top_markers()` requires an exact match to `model.cell_types`.

Recovery:

- Print `list(model.cell_types)` before choosing the label.
- Use exact capitalization and punctuation.
- If the label was removed by subsetting, reload the original model or choose a retained class.

## Conversion map file not found

Symptoms:

- `No such file: <map_file>`.
- A custom map name is accepted in one environment but not another.

Cause:

- CellTypist first checks the provided path; if absent, it looks for a bundled sample-data file with that name.
- Only CellTypist's packaged sample files are searchable by simple filename.

Recovery:

- Use an existing bundled filename such as `GENCODEv44_Gene_id2name.csv` when appropriate.
- For a custom map, pass a path that exists in the current working environment.
- Ensure the file has two comma-separated columns by default, or pass the correct `sep`.

## Conversion direction or collapse is invalid

Symptoms:

- ``convert_from` should be either 0 or 1`.
- ``convert_to` should be either 0 or 1`.
- ``convert_from` and `convert_to` should be 0 (or 1) and 1 (or 0)`.
- `Unrecognized `collapse` value`.

Cause:

- Mapping direction columns must be exactly columns 0 and 1.
- `collapse` is used only when `unique_only=False` and must be `average` or `random`.

Recovery:

- For gene symbol to Ensembl ID using `GENCODEv44_Gene_id2name.csv`, use `convert_from=1, convert_to=0` when the model features are symbols.
- For Ensembl ID to symbol, use `convert_from=0, convert_to=1`.
- For 1:N ortholog preservation, use `unique_only=False, collapse="average"` unless random representative genes are intentional and seeded.

## Conversion leaves too few features

Symptoms:

- Converted model has unexpectedly few genes.
- Annotation with the converted model has poor or zero feature overlap.

Cause:

- The mapping columns or species direction are wrong.
- The model feature namespace does not match either map column.
- `unique_only=True` discarded non-1:1 orthologs.

Recovery:

- Inspect `model.features[:10]` and mapping rows before conversion.
- Count overlap with each map column before choosing direction.
- If scientifically acceptable, retry from the original model with `unique_only=False, collapse="average"` to preserve 1:N orthologs.
- Do not reuse the partially converted object; reload from the original pickle.

## Subset keeps or excludes invalid cell types

Symptoms:

- `Please provide at least two cell types in keep_cell_types`.
- `Too many cell types to exclude, at least two should be retained`.
- `The following cell types are not found: ...`.
- `Cell types in keep_cell_types should not overlap with those in exclude_cell_types`.

Cause:

- CellTypist requires at least two retained classes and exact class-name matches.
- Keep and exclude sets cannot overlap.

Recovery:

- Inspect `list(model.cell_types)` before building keep/exclude lists.
- Ensure the retained set has two or more classes.
- Use either keep or exclude when possible; if using both, check `set(keep).isdisjoint(exclude)`.
- Reload the original model before retrying after a failed or unintended subset.

## Tiny model fails downstream annotation

Symptoms:

- The tiny model loads but annotation reports no feature overlap.
- Probabilities look different from a real built-in model.

Cause:

- The synthetic helper is designed for offline smoke tests, not biological annotation.
- Query data genes must match the fixture's feature names.
- CellTypist probabilities are sigmoid-transformed decision scores, and tiny synthetic models are not calibrated real references.

Recovery:

- Generate a matching synthetic query fixture with `tiny_model_factory.py --write-query-csv query.csv`.
- Use the tiny model only for shape, wiring, cache, CLI/API, and offline dependency checks.
- Use a real built-in or custom-trained model for scientific annotation; route annotation execution to [annotation-workflows](../../annotation-workflows/SKILL.md).
