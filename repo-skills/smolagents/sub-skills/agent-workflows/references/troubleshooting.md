# Agent Workflow Troubleshooting

Use this guide for agent-level failures. Route tool implementation issues to `tools-and-integrations`, model/provider setup to `model-providers`, executor sandbox/import security to `execution-and-safety`, and CLI/UI wrapper issues to `cli-and-ui`.

## Missing or Rejected Final Answer

Symptoms:

- The agent keeps retrying after appearing to answer.
- `agent.memory.steps` contains errors from final-answer checks.
- The run ends through `max_steps` and then synthesizes a final answer from logs.

Checks:

- In `CodeAgent`, the generated code must call `final_answer(value)`; printing or returning a Python value is not enough.
- In `ToolCallingAgent`, the final answer must be a single `final_answer` tool call, not mixed with other tool calls.
- `final_answer_checks` must return truthy values and accept `(final_answer, memory, agent=agent)`.
- If a check intentionally raises assertions, the error is stored in memory and the agent gets another step.

Fix pattern:

```python
for step in agent.memory.steps:
    if getattr(step, "error", None):
        print(step.step_number, step.error)
```

Relax overly strict checks, increase `max_steps`, or make the task instructions explicitly say to call `final_answer(...)`.

## Code Block or Parse Errors

Symptoms:

- `AgentParsingError` with text like `Make sure to provide correct code blobs`.
- The model output does not include the expected action delimiter or code block.

Checks:

- Keep `CodeAgent` prompt templates aligned with `code_block_tags`.
- Use `code_block_tags="markdown"` only when prompts/examples instruct Markdown Python fences.
- If `use_structured_outputs_internally=True`, ensure the model can follow the structured code response format.
- For `ToolCallingAgent`, confirm the model returns `ChatMessage.tool_calls` or supports parsing through `model.parse_tool_calls(...)`.

Fix pattern:

```python
agent = CodeAgent(
    tools=[],
    model=model,
    code_block_tags="markdown",
    use_structured_outputs_internally=False,
)
```

If parse failures continue, switch agent class: weak code-fence models often work better as `ToolCallingAgent`, while weak tool-call models may work better as `CodeAgent`.

## Unauthorized Imports in CodeAgent

Symptoms:

- Code execution fails with an import-not-allowed error.
- Logs warn to pass the import under `additional_authorized_imports`.

Checks:

- Add only the needed package/module names.
- Confirm the package is actually installed in the runtime environment.
- Treat `additional_authorized_imports=["*"]` as a security decision, not a convenience default.

Fix pattern:

```python
agent = CodeAgent(
    tools=[],
    model=model,
    additional_authorized_imports=["math", "statistics"],
)
```

Executor selection and sandboxing are covered by `execution-and-safety`.

## Max-Steps Exhaustion

Symptoms:

- `RunResult.state == "max_steps_error"`.
- A final memory step has `AgentMaxStepsError("Reached max steps.")`.
- The final answer is generated by `provide_final_answer(...)` from logs rather than a normal final-answer action.

Checks:

- Inspect errors in action steps to see whether parsing, execution, tool-call, or final-check failures consumed steps.
- Increase `max_steps` only after fixing repetitive errors.
- Add `planning_interval` for tasks that need multi-step decomposition.
- Simplify the task or split it across managed agents.

Fix pattern:

```python
result = agent.run("...", return_full_result=True, max_steps=12)
print(result.state)
for step in result.steps:
    if step.get("error"):
        print(step["step_number"], step["error"])
```

## Prompt Template Key Mistakes

Symptoms:

- Initialization raises `Some prompt templates are missing from your custom prompt_templates`.
- Setting `agent.system_prompt = ...` raises an `AttributeError`.
- Managed agents disappear from the manager prompt.

Checks:

- Do not assign to `agent.system_prompt`; mutate `agent.prompt_templates["system_prompt"]` instead.
- Full custom template dictionaries must include required nested keys for `planning`, `managed_agent`, and `final_answer`.
- Preserve placeholders for tools, managed agents, authorized imports, custom instructions, task, and remaining steps when relevant.

Fix pattern:

```python
agent.prompt_templates["system_prompt"] += "\nUse concise reasoning and verify assumptions."
```

When replacing the whole prompt set, start from the default YAML loaded by the selected agent class and modify it rather than writing a sparse dictionary.

## Managed-Agent Name or Description Omissions

Symptoms:

- Initialization fails with `All managed agents need both a name and a description!`.
- Duplicate-name errors mention tools or managed agents.
- The manager never calls the intended worker.

Checks:

- Every managed agent must have `name` and `description`.
- Names must be valid Python identifiers and not reserved keywords.
- Tool names, managed-agent names, and the manager's own name must be unique at the manager level.
- The manager's system prompt must include managed-agent descriptions.

Fix pattern:

```python
worker = ToolCallingAgent(
    tools=[lookup_tool],
    model=model,
    name="lookup_agent",
    description="Looks up facts and returns short sourced summaries.",
)
manager = CodeAgent(tools=[], model=model, managed_agents=[worker])
```

## Serialization Load Failures

Symptoms:

- `from_dict()` reports `Unknown model class` or `Unknown agent class`.
- `from_folder()` cannot find `agent.json`, `prompts.yaml`, tool files, or managed-agent folders.
- Loaded agents miss callbacks or final-answer checks.

Checks:

- Use `agent.save(folder)` to create the expected bundle structure before `from_folder(folder)`.
- Only registered agent classes (`CodeAgent`, `ToolCallingAgent`) deserialize by class name.
- The model class must be present in the model registry and implement `from_dict(...)`.
- Recreate `step_callbacks` and `final_answer_checks` manually after loading.
- `from_hub(...)` requires `trust_remote_code=True` after code review.

Fix pattern:

```python
agent.save("agent_bundle")
loaded = CodeAgent.from_folder("agent_bundle")
loaded.final_answer_checks = [my_check]
```

## Memory Inspection Confusion

Symptoms:

- `get_succinct_steps()` seems to omit model prompts.
- Streaming output appears not to run.
- A continuation task forgets prior steps.

Checks:

- Use `get_full_steps()` when you need `model_input_messages`; `get_succinct_steps()` intentionally removes them.
- Consume the generator returned by `run(stream=True)`; creating it is not enough.
- Use `reset=False` for continuation; default `reset=True` clears previous steps.
- Inspect manager and managed-agent memories separately.

Fix pattern:

```python
for event in agent.run("continue", stream=True, reset=False):
    print(type(event).__name__)

print(agent.memory.get_full_steps())
```

For `CodeAgent`, `agent.memory.return_full_code()` is often the fastest way to see what code actually ran.
