# Workflow Reference

Read this when executing `llamafactory-checkpoint-convert-skill`. This reference is self-contained and assumes LLaMA-Factory is installed from a package index or public repository, not from the original extraction checkout.

## Environment

```bash
python -m pip install -U pip setuptools wheel
pip install llamafactory
python ../../scripts/check_llama_factory_env.py
python scripts/check_env.py
```

Use `--package-root <path>` only when inspecting a separately installed package tree. Do not point scripts at a private source checkout.

## Inputs To Resolve

- `<model_path>`: local model path or public model id supplied by the user.
- `<work_dir>`: user output/work directory for configs, logs, summaries, and artifacts.
- `<dataset_dir>` and `<dataset_name>`: user data or a tiny dataset generated by the root helper.
- `<output_dir>`: training/export/prediction output directory.
- Smoke scale: prefer one or two samples and one training step before a full run.

## Self-Contained Demo Data

```bash
python ../../scripts/make_demo_data.py --task sft --dataset-dir <work_dir>/data --dataset-name tiny_sft
python ../../scripts/make_demo_data.py --task dpo --dataset-dir <work_dir>/data --dataset-name tiny_dpo

python ../../scripts/make_demo_data.py --task kto --dataset-dir <work_dir>/data --dataset-name tiny_kto
```

Use only the task-specific dataset needed by this sub-skill. For multimodal tasks, create or request media files from the user and validate them with the bundled validator.

- `scripts/check_env.py`: run `python scripts/check_env.py --help`; use it for validation, config generation, launch, inspection, or handoff as its name indicates.
- `scripts/inspect_checkpoint.py`: run `python scripts/inspect_checkpoint.py --help`; use it for validation, config generation, launch, inspection, or handoff as its name indicates.

## Typical Execution Pattern

```bash
mkdir -p <work_dir>
python scripts/check_env.py
python scripts/make_convert_command.py --output <work_dir>/convert_handoff.json --task hf2dcp --hf-path <model_path> --dcp-path <output_dir>
python scripts/preflight_convert.py --command <work_dir>/convert_handoff.json
python scripts/inspect_checkpoint.py --path <model_path>
```

## Success Criteria

- Environment check prints `valid: true`, or any missing optional dependency is irrelevant to the selected backend.
- Data validators report records and no fatal schema errors.
- Config files are saved under `<work_dir>` and contain user-supplied model/data/output paths.
- Smoke execution creates the expected output, checkpoint, adapter, prediction, server PID, metric, or handoff JSON artifact.
- Logs and inspection summaries are saved beside the output artifacts.

## Troubleshooting Notes

- If a package import is missing, install the package in the active environment and rerun `scripts/check_env.py`.
- If GPU memory is tight, use LoRA/QLoRA, `max_steps: 1`, `cutoff_len: 256`, `max_samples: 1-2`, and a single visible GPU.
- If a script emits a handoff JSON, that capability is not exposed as a stable installed-package CLI; follow the handoff note instead of opening the original extraction checkout.
- Keep generated configs and logs in `<work_dir>` so the final answer can cite concrete artifact paths.
