# Contig Patterns For Motif Scaffolding

This reference explains RFdiffusion contigs for motif scaffolding, target-chain layouts, inpaint masks, and length constraints.

## Core Syntax

A contig is passed as a single-item Hydra list:

```bash
'contigmap.contigs=[10-40/A163-181/10-40]'
```

Important rules:

- Chain-prefixed spans such as `A163-181` are residues from `inference.input_pdb`.
- Numeric spans such as `10-40` are new residues generated by RFdiffusion.
- Residue ranges are inclusive.
- Segment lengths are sampled per design when a range is provided.
- A single number such as `25` is accepted as a fixed generated length and is expanded internally.
- Whitespace separates contig blocks. This matters for receptor/target blocks.
- `/0 ` means a chain break and creates a large residue-index jump.

## Basic Motif Scaffold

Pattern:

```bash
'contigmap.contigs=[Nmin-Nmax/Astart-end/Cmin-Cmax]'
```

Example:

```bash
'contigmap.contigs=[10-40/A163-181/10-40]'
```

Interpretation:

- Generate 10-40 residues before the motif.
- Keep input PDB chain `A` residues 163-181 as the motif.
- Generate 10-40 residues after the motif.
- Each design may sample a different N/C-terminal length.

Command template:

```bash
run_inference.py \
  inference.output_prefix="$OUT_DIR/motif_basic" \
  inference.input_pdb="$INPUT_PDB" \
  inference.model_directory_path="$MODEL_DIR" \
  inference.num_designs=10 \
  'contigmap.contigs=[10-40/A163-181/10-40]'
```

## Fixed Total Generated Length

Use `contigmap.length` when sampled segments must satisfy a total length window.

```bash
'contigmap.contigs=[5-30/A10-25/5-30]' contigmap.length=55-55
```

The length applies to the inpainted/generated chain length after RFdiffusion samples variable segments and includes motif residues in that generated chain. If the contig ranges cannot produce a compatible length, contig sampling exits with a length incompatibility error.

## Multiple Motif Fragments

Motifs can contain discontinuous residues or fragments from the input PDB:

```bash
'contigmap.contigs=[10-100/A1083-1083/10-100/A1051-1051/10-100/A1180-1180/10-100]'
```

Use this for active-site geometry where single residues or short spans must be held in relative positions while RFdiffusion builds connecting scaffold segments.

## Target Or Receptor Chain

A target chain can be supplied as a fixed receptor block followed by `/0 ` and then the designed chain block:

```bash
'contigmap.contigs=[A25-109/0 0-70/B17-29/0-70]'
```

Interpretation:

- `A25-109/0` is a fixed receptor/target block from the input PDB.
- The space after `0` starts the generated/inpaint chain block.
- `0-70/B17-29/0-70` builds around the motif on chain `B`.
- `contigmap.length=70-120` can constrain the generated chain while excluding the fixed receptor chain.

Target-aware command template:

```bash
run_inference.py \
  inference.output_prefix="$OUT_DIR/motif_target" \
  inference.input_pdb="$INPUT_PDB" \
  inference.model_directory_path="$MODEL_DIR" \
  inference.num_designs=10 \
  'contigmap.contigs=[A25-109/0 0-70/B17-29/0-70]' \
  contigmap.length=70-120 \
  inference.ckpt_override_path="$MODEL_DIR/Complex_base_ckpt.pt"
```

## Chain-Break Checks

Correct examples:

```bash
'contigmap.contigs=[A25-109/0 0-70/B17-29/0-70]'
'contigmap.contigs=[100-100/0 B1-150]'
```

Risky or wrong examples:

```bash
contigmap.contigs=[A25-109/0 0-70/B17-29/0-70]
'contigmap.contigs=[A25-109/0/0-70/B17-29/0-70]'
'contigmap.contigs=[A25-109/0B17-29]'
```

Why they fail:

- Unquoted list syntax can be interpreted by the shell or Hydra incorrectly.
- `/0/` keeps everything in one slash-delimited block rather than using whitespace to start the next chain block.
- Missing whitespace after `0` prevents the receptor and inpaint blocks from being parsed as intended.

## Sequence Masking

Use `contigmap.inpaint_seq` to hide sequence identity for selected input residues:

```bash
'contigmap.inpaint_seq=[A163-168/A170-171/A179]'
```

Command template:

```bash
run_inference.py \
  inference.output_prefix="$OUT_DIR/motif_inpaintseq" \
  inference.input_pdb="$INPUT_PDB" \
  inference.model_directory_path="$MODEL_DIR" \
  inference.num_designs=10 \
  'contigmap.contigs=[10-40/A163-181/10-40]' \
  'contigmap.inpaint_seq=[A163-168/A170-171/A179]'
```

RFdiffusion automatically chooses an inpaint sequence checkpoint when this override is present unless `inference.ckpt_override_path` forces another checkpoint.

## Structure Masking

Use `contigmap.inpaint_str` to hide coordinates for selected input residues while keeping them in the contig mapping:

```bash
'contigmap.inpaint_str=[B165-178]'
```

Rules:

- Masked residues must exist in the input PDB and be compatible with the contig.
- RFdiffusion chooses an inpaint-capable checkpoint automatically when `inpaint_str` is set.
- Secondary-structure-specific masks, if used, must refer to residues already covered by `inpaint_str`.

## Checkpoint Patterns

Default motif scaffold:

```bash
inference.model_directory_path="$MODEL_DIR"
```

Tiny active site:

```bash
inference.ckpt_override_path="$MODEL_DIR/ActiveSite_ckpt.pt"
```

Target-aware complex motif:

```bash
inference.ckpt_override_path="$MODEL_DIR/Complex_base_ckpt.pt"
```

Inpaint sequence/structure:

```bash
'contigmap.inpaint_seq=[A163-168/A170-171/A179]'
# or
'contigmap.inpaint_str=[B165-178]'
```

Avoid overriding an automatic inpaint checkpoint unless there is a deliberate compatibility reason.

## Output Validation

After a run, inspect:

- `$OUT_DIR/<prefix>_0.pdb`: final backbone coordinates.
- `$OUT_DIR/<prefix>_0.trb`: sampled contig, mappings, inpaint masks, and resolved config.
- `$OUT_DIR/traj/`: trajectory files when trajectories are enabled.

Validation questions:

- Does the sampled contig in the `.trb` match the intended motif and length window?
- Do `con_ref_pdb_idx` and output mapping entries include every motif residue?
- Does the output keep fixed target chains separate?
- Does motif alignment show acceptable motif placement, especially for tiny active-site motifs?
