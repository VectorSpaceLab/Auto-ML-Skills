# Tracking And Segmentation Troubleshooting

Use this matrix to recover Dipy tracking, clustering, masking, tissue, and tractometry failures without assuming downloads, visualization, or original source examples.

| Symptom | Likely cause | Concrete recovery | Validation check |
| --- | --- | --- | --- |
| `LocalTracking` says the affine should be `(4, 4)` | Missing or malformed homogeneous transform | Load or construct a full 4x4 affine; use identity only for deliberate voxel-coordinate synthetic tests | `np.asarray(affine).shape == (4, 4)` before tracking. |
| Tracking rejects step size or length settings | `step_size <= 0`, `maxlen < 1`, or `minlen > maxlen` | Choose positive spatial-unit step size and consistent length bounds | Instantiate on a tiny seed list before full run. |
| Error about affine shearing | Local tracking does not accept sheared voxel-to-point transforms | Resample/reorient data upstream or use a non-shearing affine; route resampling to `../../registration-alignment/` | Off-diagonal terms of `affine[:3, :3].T @ affine[:3, :3]` are near zero. |
| No streamlines generated | Empty seed mask, seeds outside valid tissue, overly strict threshold, missing directions, or `return_all=False` discarding short tracks | Count seeds, relax stopping threshold, verify peaks/PMF in `../../reconstruction-models/`, lower `minlen`, or probe with `return_all=True` | Nonzero seeds and at least one positive-length streamline in a tiny ROI. |
| Seeds appear outside anatomy | Seed affine does not match seed/stopping mask reference | Use the same reference/affine for seed mask and stopping map; register masks upstream if needed | Convert seed coordinates back to voxel indices and confirm nonzero mask voxels. |
| `seeds_from_mask` raises `mask cannot be more than 3d` | Passing a 4D DWI, scalar array, or non-3D object | Select or compute a 3D binary seed mask first | `mask.ndim == 3` and `np.count_nonzero(mask) > 0`. |
| Stopping criterion terminates immediately or out of image | Stopping/PVE map grid differs from seed/direction reference | Resample maps to the tracking reference before constructing stopping criteria | All tracking maps share 3D shape and affine. |
| PFT or ACT output is too short/invalid | WM/GM/CSF maps swapped, unnormalized, or misaligned; step size incompatible with voxel size | Check PVE order/ranges, affine alignment, and `average_voxel_size`; probe a few seeds | PVE arrays share shape, values are plausible, and tiny PFT/CMC probe reaches expected endpoints. |
| `target` raises points outside `target_mask` | Streamline coordinates and ROI affine/reference are inconsistent | Use the tractogram's reference affine and correct space/origin with `../../io-data/` | Voxelized streamline points fall inside ROI bounds. |
| Tractogram bbox validation fails on save/load | Streamline points fall outside declared reference image or wrong space/origin was used | Fix reference image, `StatefulTractogram` space, and origin rather than blindly disabling checks | A tiny subset loads/saves with bbox checks enabled after correction. |
| QuickBundles returns one giant cluster | Threshold too large for coordinate units or geometry is over-smoothed | Lower threshold; confirm coordinates are millimeters versus voxels; inspect centroids | Cluster count increases and sizes match expected geometry. |
| QuickBundles returns one cluster per streamline | Threshold too small, inconsistent orientation/resampling, or unsuitable metric | Raise threshold; resample with `set_number_of_points`; start with default MDF metric | Cluster count decreases while centroids remain meaningful. |
| QuickBundles metric error | Unsupported metric object or unknown metric string | Use default `metric='MDF_12points'` or a supported feature plus `AveragePointwiseEuclideanMetric` | `QuickBundles(threshold).cluster(two_streamlines)` succeeds. |
| RecoBundles recognizes nothing | Subject and model bundles are not in the same space, length filters removed candidates, or thresholds are too strict | Compare coordinate ranges, route registration to `../../registration-alignment/`, relax length/reduction/pruning thresholds | Recognized bundle length and labels are nonzero on a known-overlap subset. |
| RecoBundles is slow or memory heavy | Whole-brain clustering/SLR with too many streamlines | Probe with length filters, random subset, lower `slr_select`, coarser clustering, or precomputed cluster map | Memory/runtime remain bounded while recognition signal persists on subset. |
| LabelsBundles extracts the wrong streamlines | Labels were generated from a different tractogram ordering or space | Pair labels with the exact original tractogram used by RecoBundles; keep label files and source tractogram provenance together | Label indices are within original streamline count and sampled labeled streamlines match expected anatomy. |
| `median_otsu` says 4D input needs `vol_idx` | 4D volume passed without selecting masking volumes | Provide b0-like `vol_idx`, or use CLI b-values to select b0 volumes | Returned mask is 3D and non-empty. |
| Brain mask is empty or covers non-brain | Wrong volumes, low contrast, intensity scaling, or unsuitable median/dilation settings | Use b0-like volumes; tune `median_radius`, `numpass`, `dilate`; try `finalize_mask=True` | Mask shape matches input spatial shape and overlaps expected anatomy. |
| Output shape changes unexpectedly | Deprecated `autocrop` cropped to mask bounding box | Avoid `autocrop` unless shape change is the explicit goal | Output shape is preserved when `autocrop=False`. |
| `dipy_brain_mask --method synthseg` or `evac` fails | Optional PyTorch/model weights are absent or unavailable | Use `--method median_otsu` or install/verify optional neural-network dependencies outside this skill | Brain-mask CLI help/import and a tiny median-Otsu fallback succeed. |
| HMRF classification is unstable or class labels look wrong | Wrong class count, noisy/flat image, beta/tolerance poorly tuned, or background class misunderstood | Normalize/inspect image, probe lower iterations, tune `beta`, remember implementation adds background then returns non-background PVE maps | Segmentation labels and PVE shape/value ranges are plausible. |
| `afq_profile` raises empty bundle | No streamlines survived upstream tracking/segmentation/filtering | Diagnose upstream bundle count; run profile only after non-empty bundle extraction | `len(bundle) > 0` before calling. |
| AFQ/BUAN profiles look reversed or inconsistent | Streamlines are not oriented consistently | Orient with `orient_by_streamline` or pass `orient_by` to `afq_profile` | Profile remains stable under controlled streamline reversal. |
| BUAN profiles fail on folder layout | Expected model/registered/original/metric directories are missing or group layout is wrong | Check whether the workflow is single-subject or patient/control group mode and name folders consistently | The command sees model bundles, registered bundles, original bundles, and metric volumes. |
| BUAN LMM fails with small datasets | Per-disk dataframe has too few rows for mixed models | Use more subjects/valid profiles or reduce expectations to profile arrays instead of group inference | Each disk has enough records before fitting. |
| Plots or visualization fail | FURY, matplotlib, OpenGL, display backend, or plotting dependencies are absent | Prefer numeric checks; make plots optional and environment-dependent | Smoke tests pass without importing visualization modules. |

## Boundary Routing

- PAM files, peaks, ODFs, SH coefficients, direction model fitting, or `peaks_from_model`: route to `../../reconstruction-models/`.
- Tractogram formats, `StatefulTractogram`, reference images, `Space`, `Origin`, bvals/bvecs, or bbox checks: route to `../../io-data/`.
- Transform estimation, affine/SyN registration, SLR strategy, or bundlewarp: route to `../../registration-alignment/`.
- Console entry-point discovery, parser mechanics, `--help` probing, and generic CLI flow behavior: route to `../../cli-workflows/`.
