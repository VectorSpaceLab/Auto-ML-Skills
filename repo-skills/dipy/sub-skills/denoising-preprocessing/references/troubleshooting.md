# Denoising And Preprocessing Troubleshooting

Use this matrix to recover Dipy denoising and preprocessing failures without assuming source checkout access, downloads, visualization, or optional neural-network dependencies.

| Symptom | Likely cause | Concrete recovery | Validation check |
| --- | --- | --- | --- |
| `PCA denoising can only be performed on 4D arrays` | `localpca` or `mppca` received a 2D/3D image or volumes are not on the last axis | Route file loading/axis repair to `../../io-data/`; ensure DWI shape is `(X, Y, Z, N)` | `data.ndim == 4` and `data.shape[-1] == len(bvals)` when gradients are present. |
| `gtab must be provided if sigma is not given` in `localpca` | LPCA was asked to estimate sigma without gradient metadata | Pass a scalar/3D sigma, or build `GradientTable` from bvals/bvecs through `../../io-data/` | `sigma is not None` or `gtab.bvals.size == data.shape[-1]`. |
| `Array 'arr' is incorrect shape` in PCA | Spatial dimensions are too small for `patch_radius` | Lower `patch_radius`, crop less aggressively, or use a larger data block | Every non-singleton spatial axis is at least `2 * patch_radius + 1`. |
| Warning says local PCA patch has fewer samples than volumes | Patch sample count is too low for the number of DWI volumes | Increase `patch_radius`, use a larger ROI, reduce expectations for the smoke check, or explicitly document `suppress_warning=True` only if accepted | Local sample count is greater than the number of volumes or the warning is justified. |
| `patch_radius should have length 3` or `patch_radius must be scalar or a 3-element array` | Patch radius passed as a two-value or four-value sequence | Pass an int or `(rx, ry, rz)` | Normalized patch radius has exactly three elements. |
| Patch2Self v3 rejects nonzero patch radius | `version=3` does not support spatial patch radius | Use `patch_radius=(0, 0, 0)` for v3, or switch to `version=1` if nonzero patch radius is required and runtime/dependencies are acceptable | Patch2Self preflight passes `_validate_inputs` rules. |
| Patch2Self warns about fewer than 10 volumes | Dataset is too small for reliable self-supervised denoising | Treat as smoke-only, choose LPCA/NLMeans if appropriate, or use more volumes | `data.shape[-1] >= 10` for production Patch2Self. |
| Patch2Self model error about `ols`, `ridge`, or `lasso` | Unsupported string or estimator object | Use `model='ols'`, `'ridge'`, `'lasso'`, or a scikit-learn estimator-like object in environments with scikit-learn | A tiny `patch2self` call succeeds on a small 4D array. |
| Patch2Self temporary directory error | `version=3` `tmp_dir` does not exist, or `version=1` received `tmp_dir` | Create the directory before v3, or leave `tmp_dir=None`; for v1, do not pass `tmp_dir` | `tmp_dir is None` for v1; v3 tmp dir exists if provided. |
| NLMeans says sigma length does not match the last dimension | 4D data received a 1D sigma vector with the wrong length | Use scalar sigma, `np.full(data.shape[-1], value)`, or a 3D sigma map | For 1D sigma, `sigma.shape[0] == data.shape[-1]`. |
| NLMeans says 3D sigma does not match first three dimensions | Spatial sigma map shape differs from image shape | Recompute sigma on the same image grid or resample only with explicit IO/alignment handling | `sigma.shape == data.shape[:3]`. |
| NLMeans says mask needs to be a 3D ndarray | 2D mask, 4D DWI, or wrong mask object passed | Select/build a 3D boolean mask; route brain masking to `../../tracking-segmentation/` when needed | `mask.ndim == 3` and `mask.shape == arr.shape[:3]`. |
| NLMeans output is oversmoothed | Sigma too high, `patch_radius`/`block_radius` too large, or wrong Rician/Gaussian assumption | Re-estimate sigma, reduce radii, compare `rician=True` versus `False` on a small ROI | Residual is plausible and anatomy/edges are not erased. |
| `estimate_sigma` rejects `N` | Unsupported coil correction factor | Use `N=0` for no correction or one of the supported correction values | `N` belongs to the supported set and sigma is finite. |
| `estimate_sigma` returns surprising zeros or NaNs | Mask/background handling is wrong or input is constant/empty | Re-check `disable_background_masking`; use nonzero/noisy regions; consider PIESNO if background assumptions hold | Sigma values are finite and nonnegative; mask has usable voxels. |
| PIESNO fails or produces implausible sigma | Data lacks noisy background or repeated measurements on last axis | Use `estimate_sigma` or PCA noise estimation instead; do not use PIESNO on structural-only data | PIESNO assumptions are explicitly satisfied. |
| Gibbs removal says data must be 2D, 3D, or 4D | 1D vector, 5D array, or wrong slicing/stacking | Select a valid image or collapse extra dimensions before calling | `vol.ndim in (2, 3, 4)`. |
| Gibbs removal rejects `slice_axis` | Axis is larger than 2 | Use one of the first three spatial axes; do not point at the diffusion-volume axis | `slice_axis in (0, 1, 2)`. |
| Gibbs removal modifies the input unexpectedly | Default `inplace=True` was used | Pass `inplace=False` for probes or when the original array is still needed | `np.shares_memory` behavior is intentional; output shape is unchanged. |
| Gibbs multi-processing fails or hangs in a constrained environment | `num_processes` too high, invalid, or process spawning unsupported | Use `num_processes=1` for smoke checks; avoid `0`; use `-1` only for explicit full-run approval | A single-process tiny call succeeds first. |
| Bias correction says method must be `poly`, `bspline`, or `auto` | API received an invalid method | Choose `poly` for smooth bias, `bspline` for flexible smooth bias, or `auto` to compare | Tiny method-specific call succeeds. |
| Bias correction fails because no b0 volume is available | `gtab.b0s_mask` is empty or b0 threshold is wrong | Rebuild `GradientTable` with correct b0 threshold or use a method not requiring b0-based DWI bias estimation | `gtab.b0s_mask.sum() > 0`. |
| Bias correction output is all zero outside mask | Expected behavior: corrected data is masked | Expand/fix the mask if too restrictive; use `zero_background` only for bias-field background semantics | Mask coverage is plausible and in-mask corrected values are finite. |
| Bias field has extreme values in the background | Bias is extrapolated outside the mask by default | Use `zero_background=True` when saving/QA wants field of `1.0` outside mask | `bias[~mask] == 1.0` when zero-background is enabled. |
| `dipy_correct_biasfield --method n4` fails or fetches assets | DeepN4 is optional and may need ML dependencies/model weights | Use DWI-oriented `poly`, `bspline`, or `auto`, or explicitly prepare optional NN environment outside this skill | CLI help and dependency probes confirm NN support before n4. |
| `dipy_evac_plus` fails | Optional PyTorch/TensorFlow/model assets are absent | Route general brain masks to `../../tracking-segmentation/` and use classical mask workflows unless NN dependencies are prepared | Brain mask fallback produces a nonempty 3D mask. |
| CLI writes outputs in the wrong place or refuses overwrite | Missing `--out_dir`, existing output, or workflow force/overwrite behavior | Use a fresh output directory and explicit output filenames; check `--help` for installed options | Expected output files exist only in the chosen directory. |
| CLI option spelling differs from API keyword | Workflow wrappers expose command-specific names such as `--ver`, `--skip_b0_denoising`, or `--return_sigma` | Run `command --help` and route parser details to `../../cli-workflows/` | Help output confirms options before production run. |
| `import dipy` fails from a source checkout with missing version metadata | Unbuilt checkout shadows installed package | Run scripts from outside the checkout, install the package normally, or use a prepared inspection/runtime environment | `python -c "import dipy; print(dipy.__version__)"` succeeds. |
| Visualization examples fail | `matplotlib`, `fury`, OpenGL, or display backend is absent | Skip plots; validate numerically with shapes, finite values, sigma maps, and residual summaries | Smoke tests pass without importing visualization modules. |

## Boundary Routing

- NIfTI loading/saving, bval/bvec parsing, `GradientTable` construction from files, affine/header decisions, and file conversion: route to `../../io-data/`.
- Tensor/CSD/DKI/ODF/peaks fitting after preprocessing: route to `../../reconstruction-models/`.
- Brain masks, tissue classification, stopping criteria, segmentation, and tractography: route to `../../tracking-segmentation/` unless the mask is only a preprocessing ROI.
- CLI parser mechanics, command inventory, and generic help/probe behavior: route to `../../cli-workflows/`.
