# CLAM WSI Preprocessing Workflows

CLAM's current preprocessing path is the fast coordinate pipeline: `create_patches_fp.py` segments tissue and writes one `.h5` coordinate bag per slide under `patches/`. It does not save patch images by default; later feature extraction reads coordinates and WSI pixels on demand.

## One-Pass Segmentation, Patching, and QC Stitching

Use this when the user has a source directory of WSIs and accepts default or preset segmentation parameters.

```bash
python create_patches_fp.py \
  --source DATA_DIRECTORY \
  --save_dir RESULTS_DIRECTORY \
  --patch_size 256 \
  --step_size 256 \
  --patch_level 0 \
  --preset bwh_biopsy.csv \
  --seg --patch --stitch
```

Required path flags:

- `--source`: directory containing slide files such as `.svs`, `.ndpi`, `.tif`, or `.tiff` that OpenSlide can read.
- `--save_dir`: output root that CLAM creates or updates.

Common processing flags:

- `--seg`: run HSV tissue segmentation and save mask QC images.
- `--patch`: write coordinate `.h5` files under `patches/`.
- `--stitch`: create downsampled stitched QC images from coordinate `.h5` files when they exist.
- `--patch_size`: patch width and height in pixels at `--patch_level`; CLAM examples use `256`.
- `--step_size`: coordinate stride; set equal to `--patch_size` for non-overlapping patches.
- `--patch_level`: OpenSlide pyramid level to patch; default `0` means highest resolution.
- `--preset`: a CSV template name available to CLAM, such as `bwh_biopsy.csv`, `bwh_resection.csv`, or `tcga.csv`.

Expected output layout:

```text
RESULTS_DIRECTORY/
  masks/                  # per-slide segmentation QC images, .jpg in current source
  patches/                # per-slide coordinate bags, slide_id.h5
  stitches/               # optional stitched QC images, .jpg in current source
  process_list_autogen.csv
```

`process_list_autogen.csv` records every slide, processing status, and the segmentation/filter/visualization/patch settings actually used. Keep it with the outputs because downstream feature extraction commonly starts from the `patches/` directory and an edited slide list derived from this run.

## Two-Pass Mask Review

Use this when tissue masks need review or per-slide tuning before expensive patch/stitch work.

1. Segment only:

   ```bash
   python create_patches_fp.py \
     --source DATA_DIRECTORY \
     --save_dir RESULTS_DIRECTORY \
     --patch_size 256 \
     --step_size 256 \
     --patch_level 0 \
     --preset bwh_biopsy.csv \
     --seg
   ```

2. Copy `RESULTS_DIRECTORY/process_list_autogen.csv` to a new name, for example `process_list_review.csv`, before editing. The autogenerated file is rewritten on later runs.
3. Review `masks/` images. In the copied process list, set `process=1` only for slides to rerun, tune columns such as `sthresh`, `mthresh`, `close`, `a_t`, `a_h`, `max_n_holes`, `keep_ids`, `exclude_ids`, `vis_level`, `line_thickness`, `use_padding`, or `contour_fn`, and keep other slides at `process=0`.
4. Rerun segmentation for selected slides:

   ```bash
   python create_patches_fp.py \
     --source DATA_DIRECTORY \
     --save_dir RESULTS_DIRECTORY \
     --patch_size 256 \
     --step_size 256 \
     --patch_level 0 \
     --process_list process_list_review.csv \
     --seg
   ```

5. When masks are acceptable, set `process=1` for slides that need coordinate files and run patch/stitch:

   ```bash
   python create_patches_fp.py \
     --source DATA_DIRECTORY \
     --save_dir RESULTS_DIRECTORY \
     --patch_size 256 \
     --step_size 256 \
     --patch_level 0 \
     --process_list process_list_review.csv \
     --seg --patch --stitch
   ```

`--process_list` is a filename resolved inside `--save_dir`, not a path beside the source slides. Use `--no_auto_skip` only when existing `patches/<slide_id>.h5` files should be overwritten; without it, CLAM marks existing coordinate bags as `already_exist` and skips them.

For an intentional coordinate-bag rerun after editing mask/patch settings, make the overwrite behavior explicit:

```bash
python create_patches_fp.py \
  --source DATA_DIRECTORY \
  --save_dir RESULTS_DIRECTORY \
  --patch_size 256 \
  --step_size 256 \
  --patch_level 0 \
  --process_list process_list_review.csv \
  --seg --patch --stitch \
  --no_auto_skip
```

## Preset Authoring Workflow

From this reference directory, use `../scripts/build_preset_template.py` to create a portable preset CSV before telling the user to pass it with `--preset`.

```bash
python ../scripts/build_preset_template.py \
  --preset_name custom_biopsy.csv \
  --output custom_biopsy.csv \
  --sthresh 12 \
  --mthresh 11 \
  --close 2 \
  --a_t 5 \
  --a_h 2 \
  --max_n_holes 4 \
  --line_thickness 80 \
  --contour_fn center
```

Then place or copy the resulting CSV where the CLAM command can find preset files and run:

```bash
python create_patches_fp.py \
  --source DATA_DIRECTORY \
  --save_dir RESULTS_DIRECTORY \
  --patch_size 256 \
  --step_size 256 \
  --patch_level 0 \
  --preset custom_biopsy.csv \
  --seg --patch --stitch
```

## Safe Command Builder

From this reference directory, use the bundled builder for planning, documentation, or review without launching WSI processing:

```bash
python ../scripts/clam_command_builder.py \
  --source DATA_DIRECTORY \
  --save_dir RESULTS_DIRECTORY \
  --patch_size 256 \
  --step_size 256 \
  --patch_level 0 \
  --preset bwh_biopsy.csv \
  --seg --patch --stitch
```

The helper prints the CLAM command, expected output paths, skip behavior, and safety notes. It does not import CLAM, OpenSlide, pandas, or h5py and does not read slide files.
