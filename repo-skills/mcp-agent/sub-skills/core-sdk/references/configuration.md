# Configuration and Secrets

Use this reference when building a local `mcp-agent` app from YAML files or programmatic `Settings`. Keep runtime defaults and secrets separate so failures are easy to diagnose and credentials are not embedded in reusable code.

## Settings Entrypoints

`MCPApp(settings=...)` accepts three forms:

```python
MCPApp(name="demo")                         # auto-discover settings
MCPApp(name="demo", settings="config.yaml") # explicit YAML path
MCPApp(name="demo", settings=Settings(...))  # programmatic model
```

Prefer programmatic `Settings` for tests and credential-free smoke checks. Prefer YAML files for reusable applications and CLIs.

## Discovery and Precedence

`get_settings()` resolves settings in this practical order:

1. `MCP_APP_SETTINGS_PRELOAD`: if set, parse the environment variable as a complete YAML settings document and return it directly.
2. `MCP_APP_SETTINGS_PRELOAD_STRICT=true`: if preload parsing fails, raise instead of falling back.
3. Explicit config path passed to `get_settings(config_path=...)` or `MCPApp(settings="...")`.
4. Discovered config file named `mcp-agent.config.yaml` or `mcp_agent.config.yaml` from current directory upward, each directory's `.mcp-agent/` subdirectory, then home `.mcp-agent/`.
5. Secrets file named `mcp-agent.secrets.yaml` or `mcp_agent.secrets.yaml`, first beside the config file, otherwise discovered by the same search logic.
6. Pydantic environment variables and `.env` values for `Settings` fields.

Important details:

- Config and secrets YAML are deep-merged; secrets override matching config keys.
- Lists are usually replaced by the override. Exceptions include `otel.exporters` and `workflow_task_modules`, which are concatenated and deduplicated.
- Preload is authoritative: when `MCP_APP_SETTINGS_PRELOAD` succeeds, other files and environment overrides are not applied.
- `get_settings()` caches a global settings object by default. Use `get_settings(set_global=False)` or pass `Settings(...)` explicitly when global state is undesirable.

## Minimal YAML Pair

`mcp_agent.config.yaml`:

```yaml
name: local_research
execution_engine: asyncio
logger:
  transports: [console]
  level: info
  progress_display: true
mcp:
  servers:
    fetch:
      command: uvx
      args: [mcp-server-fetch]
openai:
  default_model: gpt-4o-mini
```

`mcp_agent.secrets.yaml`:

```yaml
openai:
  api_key: ${OPENAI_API_KEY}
```

Keep the secrets file out of source control. Environment variables may be referenced inside YAML with `${VAR_NAME}` syntax and may also be read directly by provider settings.

## Programmatic Settings

Programmatic settings avoid disk discovery and are ideal for local tests, notebooks, examples, and smoke scripts:

```python
from mcp_agent.config import (
    Settings,
    LoggerSettings,
    MCPSettings,
    MCPServerSettings,
    OpenAISettings,
)

settings = Settings(
    execution_engine="asyncio",
    logger=LoggerSettings(transports=["console"], level="info"),
    mcp=MCPSettings(
        servers={
            "fetch": MCPServerSettings(
                command="uvx",
                args=["mcp-server-fetch"],
            )
        }
    ),
    openai=OpenAISettings(default_model="gpt-4o-mini"),
)
```

Do not put real API keys in reusable programmatic examples. Let callers supply secrets through environment variables, a secrets YAML file, or a preload generated by a secret manager.

## MCP Server Settings

Common server fields:

```yaml
mcp:
  servers:
    filesystem:
      transport: stdio
      command: npx
      args: ["-y", "@modelcontextprotocol/server-filesystem", "."]
      allowed_tools: [read_file, list_directory]
    remote_api:
      transport: streamable_http
      url: "https://example.invalid/mcp"
      headers:
        Authorization: "Bearer ${REMOTE_API_TOKEN}"
      http_timeout_seconds: 30
      read_timeout_seconds: 120
```

Guidance:

- `server_names` on `Agent` must match keys such as `filesystem` or `remote_api`.
- `allowed_tools` restricts which tools are exposed from that server before request-time `RequestParams.tool_filter` is applied.
- `roots` entries must use `file://` URIs.
- For OAuth-protected downstream servers, use `mcp.servers.<name>.auth.oauth` and global `oauth` token-store settings.
- Avoid embedding local absolute paths in reusable examples; use relative project paths or build them in caller code.

## Provider Settings and Secrets

Top-level provider settings include `openai`, `anthropic`, `azure`, `google`, `bedrock`, `cohere`, and `lm_studio`. Put stable defaults in config and secrets in secrets/environment:

```yaml
# config
openai:
  default_model: gpt-4o-mini
  base_url: "https://api.openai.com/v1"
anthropic:
  default_model: claude-sonnet-4-20250514
```

```yaml
# secrets
openai:
  api_key: ${OPENAI_API_KEY}
anthropic:
  api_key: ${ANTHROPIC_API_KEY}
```

Environment aliases verified by tests include:

- OpenAI: `OPENAI_API_KEY` or `OPENAI__API_KEY`.
- Anthropic: `ANTHROPIC_API_KEY`, `ANTHROPIC__API_KEY`, and `ANTHROPIC__PROVIDER`.
- Azure: `AZURE_OPENAI_API_KEY`, `AZURE_AI_API_KEY`, `AZURE__API_KEY`, `AZURE_OPENAI_ENDPOINT`, `AZURE_AI_ENDPOINT`, or `AZURE__ENDPOINT`.
- Google: `GOOGLE_API_KEY`, `GEMINI_API_KEY`, or `GOOGLE__API_KEY`.
- Bedrock: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SESSION_TOKEN`, `AWS_REGION`, `AWS_PROFILE`, or nested `BEDROCK__...` equivalents.

Provider package vs key diagnosis:

- Import failure for a provider wrapper usually means the matching optional dependency is missing.
- Provider settings with no API key usually mean config/secrets/env are incomplete.
- Authentication and rate-limit errors usually appear only during generation.

## Environment Materialization

`Settings.env` declares environment variables that `MCPApp` materializes when initialized:

```python
Settings(env=["OPENAI_API_KEY", {"SUPABASE_URL": "https://example.invalid"}])
```

Behavior:

- String entries such as `"OPENAI_API_KEY"` load from `.env` or `.env.mcp-cloud` if present and set `os.environ` only when a value is available.
- Single-key mappings provide fallback values and are applied to `os.environ`.
- Empty env variable names are invalid.
- Secret handles beginning with `mcpac_sc_` are intentionally skipped because actual values are injected by the deployment environment.
- `.env` takes precedence over `.env.mcp-cloud` when both define the same variable.

## Agents Auto-Loading

`Settings.agents` can load `AgentSpec` definitions into `app.context.loaded_subagents` during app initialization:

```yaml
agents:
  enabled: true
  search_paths:
    - "./agents"
  pattern: "**/*.yaml"
  definitions:
    - name: reviewer
      instruction: Review code for defects.
      server_names: [filesystem]
```

Precedence:

- Inline `definitions` have highest precedence.
- Earlier `search_paths` override later ones when names duplicate.
- Loaded specs become available on `app.context.loaded_subagents` for factory helpers and workflow code.

## Schema Notes

The configuration schema covers top-level settings, MCP servers, provider blocks, logging, tracing, OAuth, authorization, Temporal, environment materialization, and subagent loading. Useful schema-backed constraints reflected by `Settings` models include:

- `execution_engine` is `asyncio` or `temporal`.
- Server `transport` is one of `stdio`, `sse`, `streamable_http`, or `websocket`.
- `MCPRootSettings.uri` and `server_uri_alias` must start with `file://`.
- `logger.level` is `debug`, `info`, `warning`, or `error`.
- OpenTelemetry exporters support strings or keyed mappings such as `- file: {path: traces.jsonl}`.
- `authorization.enabled=true` requires issuer URL, resource server URL, and expected audiences.

## Safe Config Patterns

Credential-free local app with no servers:

```python
settings = Settings(
    execution_engine="asyncio",
    logger=LoggerSettings(transports=["console"], level="warning"),
    mcp=MCPSettings(servers={}),
)
```

Local app with restricted server tools:

```python
settings = Settings(
    execution_engine="asyncio",
    mcp=MCPSettings(
        servers={
            "filesystem": MCPServerSettings(
                command="npx",
                args=["-y", "@modelcontextprotocol/server-filesystem", "."],
                allowed_tools={"read_file", "list_directory"},
            )
        }
    ),
)
```

Preload from code without writing secrets to disk:

```python
import os
from pydantic_yaml import to_yaml_str

os.environ["MCP_APP_SETTINGS_PRELOAD"] = to_yaml_str(settings)
```

## Source Script Handling

Repository maintainer utilities named `promptify.py` and `gen_schema.py` are reference-only for this sub-skill:

- `promptify.py` helps maintain documentation or prompt artifacts, not runtime app construction.
- `gen_schema.py` helps regenerate the config schema from models, not run a local app.
- Future agents should use the bundled references and `scripts/check_core_sdk.py` for app-building guidance rather than invoking maintainer utilities.
