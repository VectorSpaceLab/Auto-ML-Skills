# Transform Types, Lists, and Semantics

## Choose the Cheapest Transform Family

| Family | Examples | Use when | Cost and risk |
| --- | --- | --- | --- |
| Translation and rigid | `Translation`, `Rigid`, `QuickRigid`, `DenseRigid`, `BOLDRigid`, `antsRegistrationSyNQuick[t]`, `antsRegistrationSyNQuick[r]` | Images differ mostly by shift or rotation; smoke tests; BOLD within-subject alignment. | Fastest and easiest to debug. |
| Similarity and affine | `Similarity`, `Affine`, `AffineFast`, `BOLDAffine`, `TRSAA`, `antsRegistrationSyNQuick[a]` | Global scale, shear, or robust initialization before nonlinear registration. | Moderate. `TRSAA` is higher quality but can be much slower. |
| Elastic and SyN | `Elastic`, `ElasticSyN`, `SyN`, `SyNRA`, `SyNOnly`, `SyNCC`, `SyNabp`, `SyNBold`, `SyNBoldAff`, `SyNAggro`, `SyNLessAggro` | Nonlinear anatomical correspondence after linear alignment. | Expensive; use explicit iteration schedules and masks. |
| Script-compatible SyN | `antsRegistrationSyN[x]`, `antsRegistrationSyNQuick[x]`, `antsRegistrationSyNRepro[x]`, `antsRegistrationSyNQuickRepro[x]` | Match ANTs shell-script style presets or ask for reproducible/quick variants. | Quick variants reduce cost; Repro variants matter in deterministic mode. |
| Time-varying | `TV[n]`, `TVMSQ`, `TVMSQC` | Advanced large-deformation or time-varying velocity-field workflows. | Advanced and expensive; often needs separate initialization. |

For `antsRegistrationSyN[...]`, subtype letters are: `t` translation, `r` rigid, `a` rigid plus affine, `s` rigid plus affine plus SyN, `sr` rigid plus SyN, `so` SyN-only, `b` rigid plus affine plus B-spline SyN, `br` rigid plus B-spline SyN, and `bo` B-spline SyN-only. Some `s` and `b` variants accept extra metric or spline parameters in bracket form.

## Speed and Quality Defaults

- For API checks, use `Translation`, `QuickRigid`, or `AffineFast` with short `aff_iterations`, shrink factors of `1`, and smoothing sigmas of `0`.
- For a bounded nonlinear first pass, use `antsRegistrationSyNQuick[s]`, `antsRegistrationSyNQuick[so]`, or `SyNOnly` with a short `reg_iterations` tuple such as `(40, 20, 0)`.
- For production nonlinear registration, increase iterations and resolution only after transform direction, masks, and interpolation are correct.
- `singleprecision=True` is the registration default and reduces memory use. Set `singleprecision=False` only when double precision is needed.
- If ANTs deterministic mode is active, non-`Repro` transform names can be rejected. Prefer `antsRegistrationSyNQuickRepro[...]` or `antsRegistrationSyNRepro[...]` when deterministic behavior is required.

## Initialization Semantics

- With `initial_transform=None`, ANTsPy computes a center-of-mass alignment for most non-deformable-only registrations.
- Deformable-only names such as `SyNOnly`, `antsRegistrationSyNQuick[so]`, and time-varying variants default to identity initialization unless an explicit initial transform is passed.
- Set `initial_transform='Identity'` to force identity initialization.
- Pass a list of transform filenames to `initial_transform` when prepending known transforms before a deformable stage.
- `restrict_transformation` only works when there are no preceding transforms, so do not combine it blindly with automatic or explicit initial transforms.

## Registration Transform Lists

`ants.registration(fixed, moving, ...)` returns `fwdtransforms` and `invtransforms`. Treat them as ordered lists of transform filenames unless `write_composite_transform=True`, in which case they can be composite `.h5` filenames.

- For image resampling from moving to fixed, use `tx["fwdtransforms"]` with `ants.apply_transforms(fixed=fixed, moving=moving, transformlist=...)`.
- For image resampling from fixed to moving, use `tx["invtransforms"]` with fixed/moving roles swapped.
- For point mapping, remember that image resampling internally maps output samples backward into the source image. `apply_transforms_to_points` directly maps coordinates, so the practical direction differs from `apply_transforms`.
- The same affine `.mat` file can appear in both lists; invert it with `whichtoinvert=True` when applying an inverse affine, rather than looking for a second inverse matrix file.

## `whichtoinvert` Rules

`whichtoinvert` must have one boolean per transform filename.

| Flag | Effect |
| --- | --- |
| `False` | Apply the transform as stored. |
| `True` | Apply a matrix transform as `[file.mat,1]`, meaning invert it at application time. |

Only matrix transform files can be inverted this way. Warp/displacement-field files cannot be inverted by setting `True`; use an inverse warp file, an inverse displacement-field estimate, or the `invtransforms` list generated by registration.

ANTsPy infers `(True, False)` only for the common two-item pattern `[affine.mat, warp.nii.gz]`. In all other cases it defaults to all `False`. For debugging, write flags explicitly.

## Image Warping vs Point Mapping

Image example:

```python
warped_moving = ants.apply_transforms(
    fixed=fixed,
    moving=moving,
    transformlist=tx["fwdtransforms"],
    interpolator="linear",
)
```

Point example for moving landmarks into fixed coordinates:

```python
def point_inverse_flags(transformlist):
    return [str(item).endswith(".mat") for item in transformlist]

fixed_points = ants.apply_transforms_to_points(
    dim=2,
    points=moving_points_df,
    transformlist=tx["invtransforms"],
    whichtoinvert=point_inverse_flags(tx["invtransforms"]),
)
```

Points must be physical coordinates in LPS+ convention. Convert voxel indices to physical points before transforming them.

## Interpolators

| Data | Recommended interpolator |
| --- | --- |
| Scalar intensity image | `linear`; use sinc options only when quality matters more than speed. |
| Label image or segmentation | `genericLabel` or `nearestNeighbor`; avoid intensity interpolators that create fractional labels. |
| Binary mask | `nearestNeighbor`, or re-threshold after linear interpolation if unavoidable. |
| Probability map | `linear` or `bSpline`, depending on expected smoothness. |
| Tensor/vector data | Pair interpolator choice with the correct `imagetype` so vectors/tensors are reoriented correctly. |

## `imagetype` Values

| Value | Meaning | Use |
| --- | --- | --- |
| `0` | Scalar image | Default for ordinary 2D/3D scalar images. |
| `1` | Spatial vector in index coordinates | Vector fields that should be rebased and reoriented in index space. |
| `2` | Diffusion tensor in index coordinates | Tensor images using tensor-aware reorientation and interpolation. |
| `3` | Time series | Required when transforming a 4D moving time series into a 3D fixed reference. |
| `4` | Multi-channel image | Vector-valued pixels that are channels rather than spatial vectors. |
| `6` | Spatial vector in physical coordinates | ANTs/ITK displacement fields and physical vector data. |

## Transform Objects vs Transform Files

- `ants.create_ants_transform`, `ants.read_transform`, and `ants.write_transform` operate on in-memory `ANTsTransform` objects.
- `ants.registration` and `ants.apply_transforms` primarily exchange transform filenames.
- `ants.compose_ants_transforms([tx1, tx2])` composes in-memory `ANTsTransform` objects and requires matching precision and dimension.
- `ants.apply_transforms(..., compose='prefix')` asks the backend to write a composite transform file and returns that filename when successful.
- `ants.average_affine_transform(...)` and `ants.average_affine_transform_no_rigid(...)` consume affine transform filenames and return an `ANTsTransform` object.

## Displacement Fields and Composite Outputs

- A displacement-field image must be a vector image with component count equal to the spatial dimension.
- `ants.transform_from_displacement_field(field)` converts a vector field to an `ANTsTransform`; `ants.transform_to_displacement_field(xfrm, ref)` samples it back on a reference domain.
- `ants.compose_displacement_fields(field_a, field_b)` and `ants.invert_displacement_field(field, initial_inverse)` operate on displacement-field images, not affine `.mat` files.
- `ants.create_jacobian_determinant_image(domain, tx, ...)` needs a deformation transform file or field. A pure affine `.mat` is not the usual input for local deformation diagnostics.
- `ants.deformation_gradient(warp, py_based=True)` is useful when an in-memory warp image is already available; the non-Python backend path may write temporary files internally.
