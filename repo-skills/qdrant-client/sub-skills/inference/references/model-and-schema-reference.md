# Model and Schema Reference

This reference summarizes qdrant-client inference model helpers, vector parameter helpers, and schema inspection behavior in version 1.18.0.

## Supported Model Catalog Methods

The sync and async clients expose class-level catalog methods backed by FastEmbed when the optional package is importable:

```python
from qdrant_client import QdrantClient

text_models = QdrantClient.list_text_models()
image_models = QdrantClient.list_image_models()
late_text_models = QdrantClient.list_late_interaction_text_models()
late_multimodal_models = QdrantClient.list_late_interaction_multimodal_models()
sparse_models = QdrantClient.list_sparse_models()
```

Expected return shapes:

- `list_text_models()` returns `{model_name: (dimension, models.Distance.COSINE)}`.
- `list_image_models()` returns `{model_name: (dimension, models.Distance.COSINE)}`.
- `list_late_interaction_text_models()` returns `{model_name: (dimension, models.Distance.COSINE)}`.
- `list_late_interaction_multimodal_models()` returns `{model_name: (dimension, models.Distance.COSINE)}`.
- `list_sparse_models()` returns `{model_name: description_dict}` because sparse models do not have one fixed dense dimension and distance pair.

If FastEmbed is not installed, these catalog methods return empty dictionaries because the FastEmbed classes are unavailable. They do not install packages or download model weights.

## `get_embedding_size()`

Use `client.get_embedding_size(model_name)` for dense text, dense image, late-interaction text, and late-interaction multimodal models that have a fixed per-vector dimension in the local FastEmbed catalogs.

```python
from qdrant_client import QdrantClient

client = QdrantClient(":memory:")
size = client.get_embedding_size("sentence-transformers/all-MiniLM-L6-v2")
```

Behavior:

- If `model_name` is omitted, the current dense helper model is used; by default that helper model is `BAAI/bge-small-en`.
- The method requires FastEmbed to be installed because it reads FastEmbed model catalogs.
- Passing a sparse model name raises `ValueError` with the meaning that sparse embeddings do not return fixed embedding size and distance type.
- Passing an unsupported model raises `ValueError` for unsupported embedding model.
- For Cloud-only inference, do not assume `get_embedding_size()` knows every Cloud provider model; use Cloud model metadata or a documented model dimension.

## Helper Model State

The legacy FastEmbed helper state is still useful for schema generation and compatibility workflows:

```python
client.set_model("sentence-transformers/all-MiniLM-L6-v2")
client.set_sparse_model("Qdrant/bm25")
```

`set_model()`:

- Validates and initializes a dense FastEmbed model unless `lazy_load=True` is used.
- Accepts `cache_dir`, `threads`, `providers`, `cuda`, `device_ids`, `lazy_load`, and model-specific keyword options.
- Treats `max_length` as deprecated and unused by FastEmbed models.
- Stores the selected dense helper model for `embedding_model_name`, `get_vector_field_name()`, and `get_fastembed_vector_params()`.

`set_sparse_model()`:

- Accepts a supported sparse model name or `None`.
- Initializes the sparse model when a name is provided unless `lazy_load=True` is used.
- Stores the selected sparse helper model for `sparse_embedding_model_name`, `get_sparse_vector_field_name()`, and `get_fastembed_sparse_vector_params()`.

New code can skip `set_model()` entirely and place explicit `models.Document(..., model=...)` objects in regular methods. Use helper state mainly when the collection should follow qdrant-client's helper-generated vector names.

## Dense Vector Parameter Helper

`get_fastembed_vector_params()` creates a named-vector config for the current helper dense model.

```python
vectors_config = client.get_fastembed_vector_params(
    on_disk=False,
    quantization_config=None,
    hnsw_config=None,
)
```

Return shape:

```python
{
    "fast-all-minilm-l6-v2": models.VectorParams(
        size=384,
        distance=models.Distance.COSINE,
        on_disk=False,
        quantization_config=None,
        hnsw_config=None,
    )
}
```

The exact field name is generated by `get_vector_field_name()` as `fast-` plus the final segment of `embedding_model_name`, lowercased. For `sentence-transformers/all-MiniLM-L6-v2`, the helper field is `fast-all-minilm-l6-v2`.

## Sparse Vector Parameter Helper

`get_fastembed_sparse_vector_params()` creates a sparse named-vector config for the current helper sparse model, or returns `None` when no sparse model is selected.

```python
client.set_sparse_model("Qdrant/bm25")
sparse_vectors_config = client.get_fastembed_sparse_vector_params(on_disk=False)
```

Return shape:

```python
{
    "fast-sparse-bm25": models.SparseVectorParams(
        index=models.SparseIndexParams(on_disk=False),
        modifier=models.Modifier.IDF,
    )
}
```

The exact sparse field name is generated by `get_sparse_vector_field_name()` as `fast-sparse-` plus the final segment of `sparse_embedding_model_name`, lowercased. For qdrant-client's known IDF sparse models, the helper applies `models.Modifier.IDF` unless a modifier is explicitly supplied.

## Manual Config for Stable Schemas

Use manual vector names when a collection schema must stay stable after model changes:

```python
from qdrant_client import models

vectors_config = {
    "dense": models.VectorParams(size=384, distance=models.Distance.COSINE),
}
sparse_vectors_config = {
    "sparse": models.SparseVectorParams(modifier=models.Modifier.IDF),
}
```

Then pass the same names with `using="dense"`, `using="sparse"`, or vector dictionaries such as `{"dense": models.Document(...), "sparse": models.Document(...)}`.

## Inference Input Classes

`models.Document` fields:

- `text`: text used as the embedding input.
- `model`: model name used by local FastEmbed or by the Cloud inference provider.
- `options`: optional model/provider parameters passed through as-is.

`models.Image` fields:

- `image`: image data. Cloud inference requires a URL or base64-encoded string. Local FastEmbed image models use FastEmbed-supported image input types.
- `model`: image model name.
- `options`: optional model/provider parameters.

`models.InferenceObject` fields:

- `object`: provider-specific input object.
- `model`: model name.
- `options`: optional provider/model parameters.

For local inference, qdrant-client resolves `InferenceObject` to a `Document` when the model is supported by text, sparse, or late-interaction text FastEmbed families, and to `Image` when the model is supported by image FastEmbed. It raises for late-interaction multimodal models that do not support the generic interface and for unknown model names.

## Schema Parser and Inspectors

qdrant-client detects nested inference inputs with two inspectors:

- `qdrant_client.embed.type_inspector.Inspector` answers whether a model or iterable contains any inference object.
- `qdrant_client.embed.embed_inspector.InspectorEmbed` returns field paths that should be replaced with embedded vectors.

Both inspectors use `ModelSchemaParser`, which parses generated pydantic model JSON schemas and caches paths to `Document`, `Image`, and `InferenceObject`. A bundled inspection cache covers common models and avoids reparsing every schema from scratch.

Models with inference-aware paths include:

- `models.PointStruct`, `models.PointVectors`, `models.Batch`, `models.PointsBatch`, and `models.PointsList` for writes.
- `models.NearestQuery`, `models.RecommendQuery`, `models.DiscoverQuery`, `models.ContextQuery`, and recursive `models.Prefetch` for queries.
- `models.QueryRequest`, `models.QueryRequestBatch`, `models.QueryGroupsRequest`, and update-operation wrapper models.
- `models.RecommendInput`, `models.DiscoverInput`, and `models.ContextPair` for nested positive, negative, target, and context vectors.

Plain numeric vectors, numpy arrays, sparse vectors, point IDs, `OrderByQuery`, `FusionQuery`, and `SampleQuery` are not inference inputs by themselves.

## Where Embedding Happens in Regular Methods

When `cloud_inference=False`, regular methods inspect request models before sending them:

- Query methods embed inference objects in `query`, `prefetch`, and nested query model fields with `is_query=True`.
- Write/update methods embed inference objects in points, vectors, batches, and update operations with `is_query=False`.
- If no inference objects are found, the original models are passed through.
- Local mode requires FastEmbed for any detected inference object and raises an import error if FastEmbed is missing.

When `cloud_inference=True`, qdrant-client does not locally embed inference objects. It forwards them to the remote Cloud service. This is why Cloud inference can work without FastEmbed but also why local `:memory:` clients cannot use `cloud_inference=True`.

## Async Parity

`AsyncQdrantClient` uses the same model input objects, catalog helpers, and parser concepts through `AsyncQdrantFastembedMixin`. Async method calls must be awaited, and async lifecycle belongs in the `async-client` sub-skill.
