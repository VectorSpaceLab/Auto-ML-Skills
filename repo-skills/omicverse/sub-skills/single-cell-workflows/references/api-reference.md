# Single-Cell API Reference

Import convention:

```python
import omicverse as ov
```

This table summarizes high-value `ov.single` entry points, the slots they consume, and their main outputs. Optional dependency notes are intentionally explicit because many backends are lazy or extra-gated.

## Annotation APIs

| API | Use for | Key parameters | Consumes | Produces / returns | Optional dependencies and notes |
| --- | --- | --- | --- | --- | --- |
| `ov.single.pySCSA(adata, foldchange=1.5, pvalue=0.05, output='temp/rna_anno.txt', model_path='', outfmt='txt', Gensymbol=True, species='Human', tissue='All', target='cellmarker', norefdb=False, ...)` | Marker-database cluster annotation | `species`, `tissue`, `model_path`, `target`, `foldchange`, `pvalue` | `obs[clustertype]`, gene names, `uns['rank_genes_groups']` or recomputable rankings | `cell_anno(...)` result table; `cell_auto_anno(..., key='scsa_celltype')` writes `obs[key]` | Uses SCSA database; pass a local `model_path` to avoid implicit downloads. Use `clustertype='leiden'`, not `cluster='leiden'`. |
| `pySCSA.cell_anno(clustertype='leiden', cluster='all', rank_rep=False)` | Run SCSA scoring per cluster | `clustertype`, `rank_rep` | cluster column, marker rankings | `pandas.DataFrame` with predicted cell types; cached as `scsa.result` | `rank_rep=True` can recalculate marker ranking when `rank_genes_groups` is absent. |
| `pySCSA.cell_auto_anno(adata, clustertype='leiden', key='scsa_celltype')` | Write SCSA labels back | `key` | SCSA result table and cluster labels | `adata.obs[key]` categorical labels | Expects cluster IDs convertible to the result table's cluster IDs. |
| `ov.single.get_celltype_marker(adata, clustertype='leiden', log2fc_min=2, pval_cutoff=0.05, rank=True, unique=True, global_unique=False)` | Extract marker genes by cluster | `clustertype`, thresholds | cluster labels; optionally ranking result | `dict[cluster -> genes]` | Useful before `gptcelltype` or `CellVote`. |
| `ov.single.gptcelltype(input, tissuename=None, speciename='human', ...)` | LLM-assisted marker-to-label mapping | tissue/species context | marker dict/table | cluster label mapping | Requires user-managed model/API configuration. Do not embed credentials. |
| `ov.single.get_cluster_celltype(cluster_celltypes, cluster_markers, species, organization, model, base_url, provider, api_key=None, timeout=30, max_retries=2, retry_backoff=1.5, verbose=True)` | Resolve final labels from candidate annotations and markers | candidate labels, markers, provider/model | marker and candidate dicts | `dict[cluster -> label]` | Reads `AGI_API_KEY` if `api_key` is omitted. Provider presets include OpenAI, Kimi, and Qwen endpoints. |
| `ov.single.CellVote(adata).vote(clusters_key, cluster_markers, celltype_keys, model='gpt-3.5-turbo', species='human', organization='stomach', provider='openai', result_key='CellVote_celltype')` | Consensus cell-type labels | `clusters_key`, `celltype_keys`, `cluster_markers` | cluster labels, candidate annotation columns | `obs[result_key]`, `obs[f'{result_key}_confidence']`, `uns[f'{result_key}_score_table']`, and mapping return | LLM-backed arbitration needs credentials; scoring is best-effort and skipped on failure. |
| `ov.single.Annotation(adata).annotate(method='celltypist', cluster_key='leiden', **kwargs)` | Unified annotation manager | `method`, `cluster_key`, backend kwargs | query AnnData; backend-specific resources | backend-specific result; predictions written to `obs`/`obsm` such as `obs[f'{method}_prediction']` | Methods include `celltypist`, `scsa`, `gpt4celltype`, `scMulan`, `MetaTiME`, `harmony`, `scVI`, `scanorama`, `TOSICA`. |
| `ov.single.AnnotationRef(adata_query, adata_ref, celltype_key='celltype')` | Reference label transfer | query/ref AnnData, reference label key | overlapping `var_names`; reference labels | `adata_new` concatenated object; `predict(...)` writes query labels | Warns when query/reference look log-normalized; recover counts when counts are required. |
| `AnnotationRef.predict(method='harmony', n_neighbors=15, pred_key=None, uncert_key=None)` | Transfer labels with integrated embedding or classifier | `method`, `n_neighbors` | preprocessed/integrated query-ref object | query AnnData with prediction and uncertainty columns | Methods include `harmony`, `scVI`, `scanorama`, and `TOSICA`. |

## Integration and Lazy Pipeline APIs

| API | Use for | Key parameters | Consumes | Produces / returns | Optional dependencies and notes |
| --- | --- | --- | --- | --- | --- |
| `ov.single.batch_correction(adata, batch_key, use_rep='scaled|original|X_pca', methods='harmony', n_pcs=50, **kwargs)` | Batch correction/integration | `batch_key`, `methods`, `n_pcs`, backend kwargs | `obs[batch_key]`; representation or counts depending backend | usually the same AnnData with integrated `obsm`; scVI and Concord may return model objects | Supported values include `harmony`, `combat`, `scanorama`, `scVI`, `scANVI`, `totalVI`, `scPoli`, `CellANOVA`, `Concord`, `cca`, `seurat_cca`. |
| `methods='cca'` / `'seurat_cca'` | Seurat-style CCA integration | `n_pcs`, optional `reference` | at least two batches in `obs[batch_key]` | `obsm['X_cca']`, `uns['cca']['X_cca']`; returns `adata` | Requires `pyccasc`/`cca_py`. One batch raises a `ValueError`; unknown `reference` raises `KeyError`. |
| `methods='scVI'` | Deep generative integration | `n_hidden`, `n_latent`, `dropout_rate`, `max_epochs`, `batch_size`, `accelerator` | raw counts, typically `layers['counts']`; `obs[batch_key]` | model object; `obsm['X_scVI']` | Requires `scvi-tools`. Kwargs are split between model init and training. |
| `methods='scANVI'` | Semi-supervised label-aware integration | `labels_key`, `unlabeled_category`, `n_latent`, training kwargs | `obs[batch_key]`, `obs[labels_key]`, counts | model object; `obsm['X_scANVI']` | Requires `scvi-tools`. Omitting `labels_key` raises `ValueError`. |
| `methods='totalVI'` | RNA + protein integration | `protein_expression_obsm_key`, `n_latent`, training kwargs | RNA counts and protein counts in `obsm[protein_expression_obsm_key]` | model object; `obsm['X_totalVI']` | Requires `scvi-tools`; missing protein key raises `ValueError`. |
| `methods='scPoli'` | scArches/scPoli conditional integration | `cell_type_keys`, `embedding_dims`, `n_epochs`, `pretraining_epochs` | counts, batch labels, optional cell type labels | model object; `obsm['X_scPoli']` | Requires `scarches`. |
| `ov.single.scanpy_lazy(adata, min_genes=200, min_cells=3, drop_doublet=True, n_genes_by_counts=4300, pct_counts_mt=25, target_sum=1e4, ..., n_pcs=50)` | Convenience QC-to-UMAP pipeline | QC thresholds, PCA/neighbors settings | raw AnnData | processed AnnData with counts layer, raw, HVGs, PCA, neighbors, Leiden, PAGA, UMAP | Crosses core preprocessing; use only when a one-call convenience pipeline is requested. |
| `ov.single.lazy_step_qc`, `lazy_step_preprocess`, `lazy_step_scale`, `lazy_step_pca`, `lazy_step_cell_cycle`, `lazy_step_harmony`, `lazy_step_scvi`, `lazy_step_select_best_method`, `lazy_step_mde`, `lazy_step_clustering`, `lazy_step_final_embeddings` | Stepwise lazy orchestration | stage-specific kwargs and optional output paths | AnnData slots from previous stages | updated AnnData, checkpoints/files depending stage | `lazy_step_scvi` requires scVI stack; route QC/preprocessing semantics to core analysis. |
| `ov.single.generate_scRNA_report(adata, output_path='scRNA_analysis_report.html', species='human', sample_key=None, template_dir=None, enable_analytics=True, analytics_id='OV-001')` | HTML scRNA report | `output_path`, `species`, `sample_key`, analytics controls | analyzed AnnData | path to generated HTML | Use `enable_analytics=False` unless the user explicitly wants analytics snippets. |

## Pseudobulk and Differential Inputs

| API | Use for | Key parameters | Consumes | Produces / returns | Notes |
| --- | --- | --- | --- | --- | --- |
| `ov.single.pseudobulk(adata, sample_col, groups_col=None, layer=None, raw=False, mode='sum', min_cells=0, min_counts=0, dtype=np.float32)` | Aggregate cells by sample and optionally group | `sample_col`, `groups_col`, `layer`, `mode`, filters | `obs[sample_col]`, optional `obs[groups_col]`, matrix/layer/raw | AnnData with rows as profiles, `obs['psbulk_n_cells']`, `obs['psbulk_counts']`, `layers['psbulk_props']` | `mode` must be `sum`, `mean`, or `median`. Use `sum` for count-based DE. Constant obs metadata is carried forward. |

## Trajectory, Fate, and Dynamic Feature APIs

| API | Use for | Key parameters | Consumes | Produces / returns | Optional dependencies and notes |
| --- | --- | --- | --- | --- | --- |
| `ov.single.Monocle(adata)` | Pure-Python Monocle2-style trajectory | chained methods | count-like `adata.X` | same AnnData with Monocle fields | External Monocle2 Python backend is imported lazily. |
| `Monocle.preprocess(min_expr=0.1, verbose=False)` | Detect genes, size factors, dispersions | thresholds | count-like expression | `self`; preprocessing flags and per-gene stats | Chaining method. |
| `Monocle.select_ordering_genes(genes=None, mean_expr_thresh=0.1, max_genes=None)` / `set_ordering_filter(genes)` | Choose ordering genes | explicit genes or dispersion thresholds | preprocessed AnnData | `self`; ordering gene state | Use `max_genes` to cap large datasets. |
| `Monocle.reduce_dimension(max_components=2, reduction_method='DDRTree', norm_method='log', method='fast', **kwargs)` | Learn trajectory reduction | `method='fast'` or `'exact'` | ordering genes and expression | `self`; reduced trajectory state | `exact` is slower but closer to R Monocle2 objective. |
| `Monocle.order_cells(root_state=None, reverse=None, root_by_column=None, root_by_value=None)` | Assign pseudotime and state | root selection | reduced trajectory | `obs['Pseudotime']`, `obs['State']` | Root choice controls pseudotime direction. |
| `ov.single.TrajInfer(adata, basis='X_umap', use_rep='X_pca', n_comps=50, n_neighbors=15, groupby='clusters')` | Multi-backend trajectory inference | `basis`, `use_rep`, `groupby` | embeddings and cluster labels | object with `adata`; `inference(...)` writes pseudotime/branch outputs | Optional `TrajInfer` symbol is gated by `torch` and `igraph`. |
| `TrajInfer.inference(method='palantir', **kwargs)` | Run Palantir, diffusion map, Slingshot, scTour, or StaVIA | `method` | origin/terminal cells, embeddings, markers | backend result or in-place AnnData updates | `palantir` computes markers and expects origin; Slingshot/StaVIA have their own extras. |
| `ov.single.PseudotimeFate(adata, pseudotime_key, groupby=None, scheme='hard', n_macrostates=10, residency_threshold=0.60, ..., connectivity_key='connectivities', distance_key='distances', seed=0)` | Terminal-state and fate probability estimation | pseudotime key, graph keys, macrostate settings | `obs[pseudotime_key]`, `obsp['connectivities']`, optional distances | `fit()` result; fate slots written into AnnData | Raises `KeyError` when pseudotime or graph is missing; raises `ValueError` for NaN/inf pseudotime. |
| `PseudotimeFate.fit(compute_fates=True)` | Run bias, PCCA-like coarse graining, fate probabilities | `compute_fates` | valid PseudotimeFate object | `PseudotimeFateResult`; `obsm['fate_probabilities']`, entropy, terminal groups | Use enough macrostates to cover expected terminal lineages. |
| `ov.single.dynamic_features(data, genes=None, pseudotime='pseudotime', groupby=None, groups=None, layer=None, use_raw=False, subsets=None, weights=None, distribution='normal', link='identity', n_splines=8, spline_order=3, grid_size=200, confidence_level=0.95, min_cells=20, min_variance=1e-8, store_raw=False, raw_obs_keys=None, key_added='dynamic_features', verbose=True)` | GAM trends along pseudotime | genes, pseudotime, grouping, layer/raw, GAM settings | pseudotime obs column and expression | `DynamicFeaturesResult`; optional `uns[key_added]` | Requires `pygam`. Groups below `min_cells` or low-variance genes are skipped. |

## Velocity, Perturbation, and GRN APIs

| API | Use for | Key parameters | Consumes | Produces / returns | Optional dependencies and notes |
| --- | --- | --- | --- | --- | --- |
| `ov.single.Velo(adata)` | Multi-step RNA velocity wrapper | methods such as `preprocess`, `moments`, `dynamics`, `cal_velocity` | velocity-ready layers and embeddings | velocity slots in AnnData | Backends may include dynamo, scvelo, GraphVelo, latentvelo, or regvelo extras. |
| `ov.single.velocity(adata, **kwargs)` | AnnData-first velocity calculation | forwarded to `Velo.cal_velocity` | velocity-ready AnnData | updated AnnData or backend return | Writes results in place. |
| `ov.single.cellrank_fate(adata, **kwargs)` | CellRank fate on velocity results | forwarded to `Velo.cellrank_fate` | velocity graph/layers | fitted GPCCA estimator; `uns['velocity_cellrank']` | Requires CellRank stack. |
| `ov.single.state_names(estimator)` | Extract state labels | estimator/container | CellRank estimator or labels | `list[str]` | Returns empty list if labels cannot be inferred. |
| `ov.single.perturb(adata, target, mode='ko', backend='auto', fold_change=2.0, grn_base=None, grn_output=True, return_delta=True, layer=None, n_propagation=3, backend_kwargs=None, copy=False)` | In-silico gene perturbation | target genes, mode, backend, layer, GRN options | expression matrix/layer; optional `uns['base_grn']` or `grn_base` | `PerturbResult` with perturbation, GRN, expression delta, trajectory shift | Backends are `auto`, `sctenifoldknk`, and `cell_oracle`; missing target genes or missing backend extras should be handled explicitly. |
| `PerturbResult.summary(top_n=10)` | Inspect affected genes | `top_n` | `delta_expr` table | top changed genes table | Useful immediate sanity check after perturbation. |
| `ov.single.SCENIC(adata, db_glob=None, motif_path=None, n_jobs=8, species=None, data_dir='./data/scenic', download=True, db_names=None)` | SCENIC GRN/regulon workflow | resource paths or `species`, `download`, `db_names` | expression, gene names, ranking DBs, motif annotations | object with `cal_grn()` and `cal_regulons()` | Requires SCENIC resource stack; `download=False` surfaces missing resource paths. Gene names must match ranking DB nomenclature. |
| `ov.single.grn(...)`, `build_correlation_network_umap_layout(...)`, `add_tf_regulation(...)`, `plot_grn(...)` | Smaller GRN/network utilities | function-specific | expression/correlation/TF mappings | graph layouts or plots | Use for lightweight GRN tasks when full SCENIC is unnecessary. |

## CNV, Metacell, Metabolism, and Milo APIs

| API | Use for | Key parameters | Consumes | Produces / returns | Optional dependencies and notes |
| --- | --- | --- | --- | --- | --- |
| `ov.single.CNV(adata, method='copykat', genome='hg20', layer=None, raw=False)` | Single-cell CNV inference | `method`, `genome`, `layer`, `raw` | raw counts for CopyKAT; genomic coordinates for inferCNV | `CNV` object; `run()` writes backend results | Methods are `copykat` and `infercnv`; optional backend packages are required. |
| `CNV.run(reference_key=None, reference_cat=None, exclude_chromosomes=('chrX','chrY','chrM'), platform=None, n_jobs=1, verbose=False, **kwargs)` | Execute CNV backend | reference labels, platform, exclusions | reference cells for inferCNV; chromosome/start/end var fields | `self` with `result` and AnnData side effects | inferCNV requires `platform='10x'` or `'smartseq2'` unless explicit cutoff is passed. |
| `ov.single.MetaCell(adata, method='seacells', use_rep='X_pca', n_metacells=None, layer=None, device='cpu', random_state=0, **kwargs)` | Metacell construction | backend, representation, target metacells | low-dimensional representation and optional layer | `fit()` writes `obs['metacell_id']`, confidence, backend schema | Methods include `seacells`, `metaq`, `supercell`, `kmeans`, `random`, `geosketch` depending extras. |
| `MetaCell.fit(**kwargs)` | Train selected backend | backend kwargs | selected representation | `self`; unified schema in AnnData | `n_metacells` defaults to `adata.n_obs // 75`. |
| `MetaCell.compute_purity(label_key='celltype')`, `compute_separation(use_rep=None)`, `compute_compactness(use_rep=None)` | Evaluate metacells | label or embedding keys | fitted metacells | `pandas.DataFrame` metrics | Use after `fit()`. |
| `ov.single.optimize_granularity(adata, method='seacells', n_metacells_grid=None, weight=0.5, optim_method='tradeoff', use_rep='X_pca', layer_lognorm=None, random_state=0, **backend_kwargs)` | Select metacell count | grid and metrics | expression and representations | `(best_n, sweep)` | Uses mcRigor; potentially expensive across a grid. |
| `ov.single.compare_metacell_backends(adata, backends=None, n_metacells=200, use_rep='X_pca', layer=None, layer_lognorm=None, eval_label='celltype', device='cpu', random_state=0, n_rigor_rep=30, **backend_kwargs)` | Benchmark backends | backends, metrics keys | same AnnData | comparison DataFrame | Default backends: `seacells`, `metaq`, `supercell`, `kmeans`, `random`, `geosketch`. |
| `ov.single.Metabolism(adata, method='scmetabolism', layer=None)` | Metabolic state inference | method and layer | normalized expression or Compass result alignment | `run()` writes `obsm['X_metabolism']` and `uns['metabolism']` | Methods: `scmetabolism`, `compass`, `scfea`; extras vary by backend. |
| `Metabolism.get(features=None)` / `Metabolism.to_obs(features)` | Retrieve or store metabolic features | feature names | `obsm['X_metabolism']`, `uns['metabolism']` | DataFrame or obs columns | Call only after `run()`. |
| `ov.single.differential_metabolism(adata, groupby, group1, group2='rest', method='wilcoxon')` | Compare metabolic features | groups and test | metabolism slots and group labels | DataFrame with `feature`, `mean1`, `mean2`, `log2fc`, `pval`, `padj`, `direction` | Methods: `wilcoxon` or `t-test`. |
| `ov.single.Milo().load(input, feature_key='rna')` | Prepare MuData for Milo | feature key | AnnData | MuData with `feature_key` and `milo` modalities | Requires `mudata`. |
| `Milo.make_nhoods(data, neighbors_key=None, feature_key='rna', prop=0.1, seed=0, copy=False)` | Define Milo neighborhoods | graph key, sampling prop | neighbor graph at `obsp['connectivities']` or named graph | `obsm['nhoods']`, `obs['nhood_ixs_refined']`, `obs['nhood_kth_distance']`, `uns['nhood_neighbors_key']` | Run neighbors first. |

## Public Symbols Also Worth Checking

The `omicverse.single` namespace also exposes `cosg`, `find_markers`, `get_markers`, `auto_resolution`, `SCCAF_assessment`, `geneset_aucell`, `pathway_aucell`, `diffmap`, `aucell`, `MetaTiME`, `scanpy_cellanno_from_dict`, `CellOntologyMapper`, `MetaboliteCCC`, `differential_metabolism`, `cell_fate_perturbation`, `perturbation_effect`, `velocity_effect`, and optional symbols such as `pySIMBA`, `pyTOSICA`, `scLTNN`, `cytotrace2`, `scDiffusion`, and `pyCEFCON`. Treat optional symbols as feature-gated and verify imports before promising them in an implementation.
