# Scenario Pages

Create one Markdown page per practical repository usage scenario in this
directory, using the shape defined in `../usage-scenarios.md`.

Scenario pages belong to the live user-managed `repo-skills-router`. Each page
is the second disclosure layer: list the repo skill options for that usage
scenario, explain each skill's concrete role, and include the comparison and
selection guideline for similar repo skills in that same page.

The package-bundled copy starts empty and acts as a template for future
imports. Live scenario pages are generated by
`verify-repo-skill/scripts/update_repo_skills_router.mjs` from
structured repo skill routing metadata; update that metadata and rerun the
updater rather than hand-editing scenario pages during import.
